## 2. 技术栈总览

### 架构模式

本系统采用 **前后端分离架构**，具体特征如下：

- **前后端分离**：前端 Vue3 应用与后端 Spring Boot 应用独立部署、独立开发
- **RESTful API**：后端提供标准化 RESTful 接口，前端通过 HTTP 请求进行数据交互
- **JWT 认证**：基于 JSON Web Token 的无状态身份验证机制
- **反向代理**：前端通过 Vite 开发服务器代理 API 请求至后端服务
- **响应式设计**：前端采用组件化开发，支持多终端适配

### 5.2 字符集配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **字符集** | UTF-8 | 支持 Unicode 全字符集 |
| **编码方式** | characterEncoding=UTF-8 | JDBC 连接字符编码 |
| **Unicode 支持** | useUnicode=true | 启用 Unicode 转换 |
| **排序规则（推荐）** | utf8mb4_general_ci 或 utf8mb4_unicode_ci | 支持 emoji 和特殊字符 |

**为什么选择 UTF-8：**
- 完整支持中文（CJK 统一汉字）
- 支持 emoji 表情符号（utf8mb4）
- 国际化文本存储需求
- 与 iText Asian 中文字体配合

### 5.3 存储引擎要求

| 存储引擎 | 适用场景 | 推荐度 |
|----------|----------|--------|
| **InnoDB**（推荐） | 事务性表、高并发读写 | ⭐⭐⭐⭐⭐ |
| MyISAM | 只读或低并发场景 | ⭐⭐ |

**InnoDB 关键特性：**
- **ACID 事务支持**：确保数据一致性
- **行级锁**：高并发性能优异
- **外键约束**：数据引用完整性
- **崩溃恢复**：自动崩溃恢复机制

**建议的建表语句模板：**

```sql
CREATE TABLE table_name (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    -- 字段定义...
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted TINYINT(1) DEFAULT 0  -- 逻辑删除字段
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 7.3 JWT 配置

```properties
# ========== JWT 令牌配置 ==========
jwt.secret=your-very-long-secret-key-that-is-at-least-64-bytes-long-for-hs512-algorithm-security
jwt.expiration-ms=86400000
```

**参数详解：**

| 参数 | 当前值 | 说明 | 安全建议 |
|------|--------|------|----------|
| jwt.secret | *64+ bytes* | HS512 签名密钥 | 🔒 生产环境必须更换为随机生成的强密钥 |
| jwt.expiration-ms | 86400000ms | Token 有效期（24小时） | 可根据业务调整（建议 2-24h） |

**JWT 工作流程：**

```
1. 用户登录 → 验证凭据
                    ↓
2. 生成 JWT Token（Payload: userId, username, roles, exp）
                    ↓
3. 返回 Token 给前端（通常放在 Response Header 或 Body）
                    ↓
4. 前端存储 Token（localStorage / sessionStorage / Cookie）
                    ↓
5. 后续请求 → Header: Authorization: Bearer <token>
                    ↓
6. 后端过滤器验证 Token（签名、有效期）
                    ↓
7. 解析用户信息 → 设置 SecurityContext → 继续业务逻辑
```

**JWT Token 结构：**
```
Header.Payload.Signature

示例:
eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiIxIiwidXNlcm5hbWUiOiJhZG1pbiIsInJvbGVzIjpbIlJPTEVfQURNSU4iXSwiaWF0IjoxNzEyMzQ1Njc4LCJleHAiOjE3MTI0MzIwNzh9.<signature>
```
