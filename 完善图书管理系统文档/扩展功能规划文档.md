# 城市图书馆管理系统 - 扩展功能规划文档

## 1. 规划概述

### 1.1 规划背景和目标

城市图书馆管理系统（CityLibrarySystem）是一个基于 **Spring Boot + Vue3 + MyBatis-Plus + MySQL** 技术栈构建的现代化图书管理平台。系统当前已实现 **28个Controller模块**、**20个Service服务层**、**25个数据模型实体**，涵盖用户认证、图书管理、借阅流程、座位预约、通知公告、统计分析等核心业务领域。

**规划目标：**
- 在现有架构基础上，渐进式扩展系统能力
- 提升用户体验和服务质量
- 增强系统的智能化和自动化水平
- 为未来智慧图书馆生态奠定技术基础

### 1.2 规划原则

| 原则 | 说明 |
|------|------|
| **渐进式演进** | 每个阶段基于前一阶段成果，避免大爆炸式重构 |
| **向后兼容** | 新功能不影响现有功能的正常运行 |
| **用户体验优先** | 所有扩展以提升用户满意度为最终目标 |
| **技术可行性** | 基于当前Spring Boot生态和Vue3生态的可扩展空间进行规划 |
| **成本效益平衡** | 优先投入产出比高的功能 |

@Service
public class CollaborativeFilteringService {

    /**
     * 基于用户的协同过滤
     * 1. 找到与目标用户相似的用户群体

    public List<Book> recommendByUserCF(Long userId, int topN);

#### 3.2.2 协同过滤算法实现

**算法选择：**

| 算法类型 | 适用场景 | 复杂度 | 冷启动问题 |
|----------|---------|--------|------------|
| **基于用户(User-CF)** | 用户数量 < 物品数量 | O(m²n) | 新用户无数据 |
| **基于物品(Item-CF)** | 用户数量 > 物品数量 | O(n²m) | 新物品无数据 |
| **矩阵分解(SVD)** | 大规模稀疏矩阵 | O(k(m+n)) | 需辅助信息 |
| **深度学习(DNN)** | 复杂特征交叉 | 高 | 需预训练 |

**推荐实施路径：**

```
阶段1: 基于规则的增强（当前）
  ↓
阶段2: 引入Item-CF（利用现有借阅数据）
  ↓
阶段3: 混合策略（内容 + 协同 + 热度）
  ↓
阶段4: 引入机器学习库（Apache Mahout / TensorFlow Java）
```
