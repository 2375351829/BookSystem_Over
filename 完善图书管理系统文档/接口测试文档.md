### 3.2 获取JWT Token

#### 登录接口调用示例

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "Admin@123"
  }'
```

#### 成功响应示例
```json
{
  "code": 200,
  "message": "Login successful",
  "accessToken": "eyJhbGciOiJIUzUxMiJ9...",
  "refreshToken": "eyJhbGciOiJIUzUxMiJ9...",
  "tokenType": "Bearer",
  "user": {
    "id": 1,
    "username": "admin",
    "realName": "管理员",
    "userType": "ADMIN",
    "role": "ROLE_ADMIN",
    "status": 1,
    "deleted": 0
  },
  "identityId": null,
  "identityType": null
}
```

#### Token使用方式
在后续请求的Header中添加：
```
Authorization: Bearer <accessToken>
```

#### Token刷新机制
当accessToken过期时，使用refreshToken获取新的accessToken：

```bash
curl -X POST http://localhost:8080/api/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{
    "refreshToken": "<refreshToken>"
  }'
```

### 4.3 Token刷新接口

**接口信息：**
- URL: `POST /api/auth/refresh`
- 认证: 不需要
- 请求参数(@RequestBody): Map<String,String> {refreshToken}
- 响应格式: code, accessToken, tokenType

**测试用例：**

| 用例ID | 用例名称 | 前置条件 | 请求参数 | 预期结果 | 优先级 |
|--------|----------|----------|----------|----------|--------|
| AUTH-REFRESH-001 | 正常刷新Token | 有效refreshToken | {"refreshToken":"valid_token"} | 返回200,新accessToken | P0 |
| AUTH-REFRESH-002 | refreshToken为空 | - | {"refreshToken":""} | 返回400,提示必填 | P0 |
| AUTH-REFRESH-003 | refreshToken无效 | - | {"refreshToken":"invalid"} | 返回401,提示无效 | P0 |
| AUTH-REFRESH-004 | refreshToken已过期 | 过期的token | {"refreshToken":"expired"} | 返回401,提示过期 | P1 |
| AUTH-REFRESH-005 | 用户已被删除 | 有效但用户不存在 | {"refreshToken":"valid_but_user_deleted"} | 返回404,用户不存在 | P1 |

**请求示例：**
```bash
POST /api/auth/refresh
Content-Type: application/json

{
  "refreshToken": "eyJhbGciOiJIUzUxMiJ9..."
}
```

**成功响应示例：**
```json
{
  "code": 200,
  "accessToken": "eyJhbGciOiJIUzUxMiJ9...(new token)",
  "tokenType": "Bearer"
}
```

### A. HTTP状态码说明

| 状态码 | 含义 | 出现场景 |
|--------|------|----------|
| 200 OK | 请求成功 | 正常的GET/POST/PUT操作 |
| 201 Created | 创建成功 | 资源创建(部分接口) |
| 204 No Content | 无内容 | 删除操作成功 |
| 400 Bad Request | 请求参数错误 | 参数校验失败、必填字段缺失 |
| 401 Unauthorized | 未认证 | 缺少Token、Token无效/过期 |
| 403 Forbidden | 无权限 | 角色权限不足 |
| 404 Not Found | 资源不存在 | ID对应的资源不存在 |
| 405 Method Not Allowed | 方法不允许 | 使用了不支持的HTTP方法 |
| 415 Unsupported Media Type | 不支持的媒体类型 | Content-Type错误 |
| 500 Internal Server Error | 服务器内部错误 | 程序异常、数据库错误 |

### B. 业务错误码汇总

| 错误码(code) | 含义 | 出现场景 |
|--------------|------|----------|
| 200 | 成功 | 操作成功完成 |
| 400 | 请求参数错误 | 校验失败、必填项缺失 |
| 401 | 未认证/登录失败 | Token无效、用户名密码错误 |
| 403 | 无权操作 | 非资源所有者、角色不足 |
| 404 | 资源不存在 | ID无对应记录 |
| 500 | 服务器内部错误 | 程序异常 |
