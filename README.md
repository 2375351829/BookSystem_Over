## 📖 项目简介

城市图书馆管理系统（CityLibrarySystem）是一个面向城市公共图书馆/高校图书馆的现代化、智能化图书管理平台。采用前后端分离架构，支持多角色（普通读者/学生/教师/国际读者/图书管理员/系统管理员）协同工作，提供完整的图书管理、借阅管理、座位预约、用户管理、数据统计等全方位功能。

### 🎯 核心价值主张

| 价值维度       | 具体说明                              |
| ----------- | ------------------------------- |
| **全流程数字化** | 从图书采编到借阅归还的全流程电子化管理，减少人工操作，提升效率       |
| **多角色权限体系** | 基于RBAC的细粒度角色权限控制，满足不同用户群体的多样化需求         |
| **智能座位预约** | 阅览室座位的在线预约、可视化选择、签到签退管理，优化座位资源配置       |
| **数据驱动决策** | 丰富的统计报表功能，支持多维度数据分析，为管理决策提供依据         |
| **多渠道通知** | 站内消息、预留短信邮件多渠道通知机制，提升用户服务体验           |
| **国际化支持** | 中文/英文双语界面，支持国际读者使用，适配多语种书籍管理           |

### 🎯 设计目标

| 目标       | 说明                              |
| -------- | ------------------------------- |
| **高可用性** | 核心业务7×24小时稳定运行，目标可用性 99.9%      |
| **高性能**  | 接口响应时间 P99 < 500ms，支持并发用户数 500+ |
| **安全性**  | 符合等级保护2.0要求，数据加密传输和存储，密码BCrypt加密           |
| **可扩展性** | 模块化设计，支持功能水平扩展和垂直扩展，插件化架构             |
| **易维护性** | 清晰的代码结构、完善的文档、自动化测试覆盖、Flyway数据库迁移           |

### 📋 项目背景

随着数字化转型的推进，传统图书馆管理系统已无法满足现代图书馆的业务需求。本项目旨在打造一个功能全面、用户友好、技术先进的图书馆管理系统，解决传统系统存在的痛点：

| 痛点 | 解决方案 | 业务价值 |
| ---- | ------ | ------ |
| **传统系统功能单一** | 整合图书管理、借阅管理、座位预约、统计报表等全方位功能 | 满足现代图书馆多元化的服务需求 |
| **用户体验差** | 采用Vue3 + Element Plus打造现代化响应式界面 | 界面美观、操作简洁、用户体验优秀 |
| **管理效率低** | 图书批量导入、自动ISBN识别、借阅自动化流程 | 减少人工操作，提升工作效率50%以上 |
| **数据孤岛** | 统一数据平台，全流程数据贯通 | 便于统一管理和深度分析 |
| **安全性不足** | JWT认证 + Spring Security + 操作日志审计 | 完善的权限控制和数据保护机制 |
| **缺乏国际化** | 中英文双语界面，支持国际读者 | 适应国际化发展趋势 |

本系统通过整合现代信息技术，为图书馆提供全方位的管理解决方案，提升图书馆的服务质量和管理效率。

### 💡 与同类产品的差异

| 特性 | 传统图书馆系统 | CityLibrarySystem |
| ---- | ---------- | ---------------- |
| **技术架构** | 传统单体架构，技术栈老旧 | 前后端分离，Spring Boot 3.x + Vue 3.x，技术先进 |
| **座位预约** | 无或功能简单 | 可视化座位选择、签到签退、违规管理，功能完整 |
| **用户类型** | 用户类型单一 | 支持普通读者/学生/教师/国际读者，差异化权限 |
| **国际化** | 仅中文 | 中英文双语，支持国际读者 |
| **数据统计** | 基础统计报表 | 多维度统计分析、ECharts可视化图表、数据导出 |
| **图书采编** | 需人工录入 | ISBN自动识别、批量导入、编目工作流 |
| **安全认证** | 传统会话认证 | JWT无状态认证、操作日志审计 |

### 🏢 适用场景

- **城市公共图书馆**：面向市民提供图书借阅、座位预约、咨询服务
- **高校图书馆**：面向师生提供教学科研文献服务、学习空间预约
- **专业图书馆**：面向特定领域提供专业文献资源服务
- **企业图书馆**：面向企业员工提供知识服务、学习空间管理

| **借阅管理** | 借阅/归还/续借、逾期管理 | 借阅规则校验、状态机、自动计算 | 图书流通管理、借阅服务 |

| **消息通知** | 系统通知、借阅提醒、站内消息 | 多渠道通知、模板管理、消息分类 | 用户服务提醒、信息推送 |
| **参考咨询** | 在线咨询、FAQ管理、意见反馈 | 智能问答、咨询分类、回复追踪 | 读者咨询服务、问题解答 |
| **罚款管理** | 逾期罚款、损坏/丢失罚款、罚款缴纳 | 罚款计算规则、减免机制、支付记录 | 违规处理、费用管理 |
| **统计报表** | 借阅统计、用户统计、热门排行 | ECharts可视化、多维度分析、数据导出 | 数据决策、业务分析 |
| **系统配置** | 借阅规则、罚款规则、系统参数 | 配置化管理、规则引擎、热更新 | 业务规则调整、系统设置 |
| **采购采编** | 采购申请、编目工作流、图书加工 | 采购流程管理、任务追踪、状态流转 | 图书采购、资源建设 |
| **评价推荐** | 图书评价、收藏夹、个性化推荐 | 评分系统、收藏管理、用户行为追踪 | 用户互动、阅读推广 |
| **操作日志** | 用户操作记录、系统变更日志 | 日志审计、操作追踪、安全监控 | 安全审计、问题追溯 |

|-----|------|
| **使用场景** | 用户管理个人借阅记录，办理续借、查看借阅历史等 |

#### 📅 我的预约

![我的预约](./展示图/4.首页-我的预约.png)

| 维度 | 说明 |
|-----|------|
| **使用场景** | 用户预约热门图书，跟踪预约状态，管理预约记录 |
| **技术特点** | 预约队列管理、状态机流转、定时任务自动处理超时预约 |
| **核心功能** | 图书预约、状态跟踪、预约取消、到馆通知 |

- **预约状态跟踪**：实时跟踪预约图书的状态，包括等待中、已通知、已取消、已失效四种状态
- **预约取消**：用户可以在图书到馆前主动取消预约，释放预约资源
- **到馆通知**：图书到馆后系统自动发送通知提醒用户，用户有7天时间办理借阅
- **预约有效期**：预约有效期为7天，超过有效期未借阅自动失效
- **预约队列**：同一本图书支持多人预约，按预约时间排序形成预约队列
- **重复预约检查**：防止用户重复预约同一本未归还的图书

#### 🪑 座位预约

![座位预约](./展示图/5.首页-座位预约.png)

| 维度 | 说明 |
|-----|------|
| **使用场景** | 用户预约阅览室座位，管理学习空间资源 |
| **技术特点** | 可视化座位布局、实时状态更新、冲突检测、签到机制 |
| **核心功能** | 可视化选择、多时段预约、签到签退、违规处理 |

- **可视化座位选择**：通过直观的座位图选择可用座位，不同颜色标识不同状态（空闲/已预约/使用中/维护中）
- **阅览室实时状态**：显示各阅览室的当前占用情况、可用座位数量、总座位数等实时数据
- **多时段预约**：支持选择不同时间段进行座位预约，精确到具体的开始和结束时间
- **座位类型选择**：根据需要选择不同类型的座位（普通座、带电源、静音区、小组讨论区等）
- **一键签到/签退**：通过扫码或手动操作完成座位的签到和签退，记录实际使用时间
- **预约提醒**：预约成功后发送提醒，临近预约时间再次提醒，防止错过签到
- **冲突检测**：自动检测同一座位同一时间段的预约冲突，防止重复预约
- **违规处理**：预约后未按时签到标记为违约，影响后续预约权限
- **座位布局管理**：支持座位布局的可视化编辑，支持多种布局模板

├── 用户管理（CRUD、批量操作、导入导出）

LIBRARIAN (图书管理员)

READER / STUDENT / TEACHER / INTERNATIONAL (读者)

## 📋 业务规则说明

### 借阅规则

| 规则项 | 默认值 | 说明 | 配置位置 |
|--------|--------|------|----------|
| 默认借阅期限 | 30天 | 从借阅日起计算应还日期 | system_config.borrow_days |
| 最大续借次数 | 2次 | 每本图书最多可续借2次 | BorrowController |
| 每次续借天数 | 30天 | 续借时延长30天借阅期限 | BorrowController |
| 最大借阅数量 | 10本 | 单用户最大在借图书数 | system_config.max_borrow_count |
| 续借上限配置 | 3次 | 系统配置中的续借次数上限 | system_config.renew_limit |

**不同用户类型的借阅规则差异**：
- 普通读者(READER)：标准规则
- 学生(STUDENT)：可享有更长期限或更多数量
- 教师(TEACHER)：可享有更长借阅期限
- 国际读者(INTERNATIONAL)：可有特殊政策

### 罚款规则

| 罚款类型 | 计算公式 | 宽限期 | 上限 | 说明 |
|----------|----------|--------|------|------|
| 逾期罚款 | 0.50元/天 × (逾期天数-3) | 3天 | 100元 | 前3天免费，之后每天0.5元 |
| 损坏罚款 | 图书原价 × 比例 | 无 | 无 | 具体比例由Service层确定 |
| 丢失罚款 | 图书原价 × 倍数 | 无 | 无 | 通常为1-3倍 |

**罚款支付流程**：
1. 系统自动计算罚款（逾期检查定时任务）
2. 用户查看未支付罚款列表
3. 用户在线缴纳或管理员代缴
4. 支付后更新paid_status=1，记录paid_date
5. 特殊情况管理员可取消罚款（需填写原因）

### 🎯 设计原则

### SOLID原则应用

#### 单一职责原则（SRP）
- 每个Controller只负责一个领域的HTTP请求处理
- 每个Service接口只定义该领域的操作
- 模块化设计，职责清晰分离

#### 开闭原则（OCP）
- 使用策略模式实现通知通道扩展
- 通过配置URL白名单控制权限，新增接口只需添加配置
- MyBatis-Plus的BaseMapper通过继承即可获得CRUD能力

#### 接口隔离原则（ISP）
- 每个Service接口小而精，只定义该领域的操作
- Controller只注入需要的Service接口
- 前端API模块按领域拆分（auth.ts, book.ts, borrow.ts等）

#### 依赖倒置原则（DIP）
- Controller → Service接口 → ServiceImpl → Mapper接口
- 通过构造函数注入依赖
- 配置类依赖抽象接口而非具体实现

### 其他设计原则

#### DRY原则（Don't Repeat Yourself）
- 统一响应结果封装（Result<T>）
- 公共代码抽取到工具类
- 配置集中管理

#### KISS原则（Keep It Simple, Stupid）
- 单体应用架构，开发部署简单
- 直接使用RESTful API，无GraphQL复杂查询
- Element Plus开箱即用，无深度定制主题

### 📡 RESTful API设计规范

#### URL命名规范
- 使用名词复数形式：`/api/books`（不是 `/api/book`）
- 全部小写：`/api/user-profiles`（不是 `/api/UserProfiles`）
- 使用连字符分隔单词：`/api/borrow-records`（不是 `/api/borrowRecords`）
- 避免层级过深（建议不超过3层）：`/api/books/{id}/reviews`

#### HTTP方法语义
| 方法 | 幂等性 | 安全性 | 语义 | 使用场景 |
|------|--------|--------|------|----------|
| **GET** | 是 | 是 | 获取资源 | 查询列表、详情 |
| **POST** | 否 | 否 | 创建资源/执行动作 | 新增、提交表单、特殊操作 |
| **PUT** | 是 | 否 | 完整替换资源 | 全量更新 |
| **DELETE** | 是 | 否 | 删除资源 | 删除数据 |

#### 统一响应格式
**成功响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": { "id": 1, "title": "图书名称" },
  "success": true
}
```

**错误响应：**
```json
{
  "code": 400,
  "message": "参数校验失败",
  "data": null,
  "success": false,
  "errors": [
    { "field": "title", "message": "不能为空" }
  ]
}
```

#### 命名规范
- **表名**：小写字母 + 下划线分隔（snake_case），如 `user_account`、`borrow_record`
- **字段名**：小写字母 + 下划线分隔（snake_case），如 `user_id`、`book_title`
- **主键**：统一使用 `id` 字段，类型为 BIGINT 自增
- **外键**：`{关联表名}_id` 格式，如 `user_id`、`book_id`
- **时间字段**：`{动作}_at` 格式，如 `created_at`、`updated_at`
- **布尔字段**：`is_{形容词}` 或 `has_{名词}` 格式，如 `is_active`、`has_borrowed`

#### 范式要求
- 遵循第一范式（1NF）：每列都是不可分割的原子值
- 遵循第二范式（2NF）：非主键字段完全依赖于主键
- 遵循第三范式（3NF）：非主键字段不传递依赖于主键
- 适当的反范式化：对于读多写少且需要频繁JOIN的场景，可适当冗余以提升查询性能

   - ✅ 配置浏览器缓存策略
   - ✅ 使用CDN加速静态资源访问

3. **监控与备份**
   - ✅ 配置系统监控（Prometheus + Grafana）
   - ✅ 定期数据库备份（建议每天备份）

## ❓ 常见问题

### 安装部署相关

**Q: 启动后端时提示数据库连接失败怎么办？**

A: 请检查以下几点：
1. 确认MySQL服务是否已启动
2. 检查数据库连接配置（用户名、密码、端口）
3. 确认已创建CityLibrary数据库
4. 检查防火墙是否阻止了数据库连接

**Q: 前端无法访问后端API怎么办？**

A: 请检查：
1. 后端服务是否正常启动（http://localhost:8080）
2. 检查CORS配置是否正确
3. 确认前端代理配置（vite.config.ts）中的后端地址是否正确
4. 查看浏览器控制台的具体错误信息

**Q: npm install失败怎么办？**

A: 可以尝试：
1. 检查Node.js版本是否满足要求（≥16.x）
2. 清除npm缓存：`npm cache clean --force`
3. 使用淘宝镜像：`npm config set registry https://registry.npmmirror.com`
4. 删除node_modules和package-lock.json，重新安装

### 业务功能相关

**Q: 如何配置借阅规则？**

A: 管理员可以：
1. 登录管理后台 → 系统管理 → 借阅规则
2. 按用户类型设置不同的借阅期限、数量限制、续借次数
3. 系统支持实时生效，无需重启服务

**Q: 逾期罚款如何计算？**

A: 逾期罚款计算规则：
1. 宽限期：前3天不收费
2. 计算公式：罚款金额 = 0.50元 × (逾期天数 - 3)
3. 罚款上限：最高100元
4. 管理员可在系统配置中调整罚款规则

**Q: 座位预约违约如何处理？**

A: 座位预约违约处理：
1. 预约后未在规定时间内签到，标记为违约
2. 违约记录会影响后续预约权限
3. 管理员可在座位管理中查看和处理违约记录
4. 可在系统配置中调整违约处理规则
