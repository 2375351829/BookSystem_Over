## 📖 项目简介

城市图书馆管理系统是一个面向城市公共图书馆/高校图书馆的现代化、智能化图书管理平台。采用前后端分离架构,支持多角色(读者/管理员/超级管理员)协同工作,提供完整的图书管理、借阅管理、座位预约、用户管理等功能。

### 🎯 设计目标

| 目标       | 说明                              |
| -------- | ------------------------------- |
| **高可用性** | 核心业务7×24小时稳定运行,目标可用性 99.9%      |
| **高性能**  | 接口响应时间 P99 < 500ms,支持并发用户数 500+ |
| **安全性**  | 符合等级保护2.0要求,数据加密传输和存储           |
| **可扩展性** | 模块化设计,支持功能水平扩展和垂直扩展             |
| **易维护性** | 清晰的代码结构、完善的文档、自动化测试覆盖           |

### 📋 项目背景

随着数字化转型的推进，传统图书馆管理系统已无法满足现代图书馆的业务需求。本项目旨在打造一个功能全面、用户友好、技术先进的图书馆管理系统，解决传统系统存在的痛点：

- **传统系统功能单一**：无法满足现代图书馆多元化的服务需求
- **用户体验差**：界面老旧，操作繁琐
- **管理效率低**：人工操作多，自动化程度低
- **数据孤岛**：各系统之间数据不互通，难以进行统一管理和分析
- **安全性不足**：缺乏完善的权限控制和数据保护机制

本系统通过整合现代信息技术，为图书馆提供全方位的管理解决方案，提升图书馆的服务质量和管理效率。

## ✨ 功能特性

### 👤 用户端功能

#### 🏠 首页

![首页](./展示图/1.首页.png)

系统首页为用户提供直观的信息入口，包含以下功能：
- **轮播图展示**：展示图书馆最新活动、推荐图书和重要通知
- **热门图书推荐**：基于用户行为和借阅数据的智能推荐
- **最新公告**：图书馆最新通知和活动信息
- **快速入口**：提供图书搜索、借阅、预约等常用功能的快捷访问
- **统计概览**：显示用户当前借阅数量、预约数量等个人统计信息

#### 📚 图书列表

![图书列表](./展示图/2.首页-图书列表.png)

- **多维度搜索**：支持书名、作者、ISBN、分类等多种搜索方式
- **高级筛选**：按分类、出版社、出版日期、热度等多维度筛选
- **图书详情**：展示图书封面、简介、作者信息、出版信息等详细内容
- **库存状态**：实时显示图书的可借数量和所在位置
- **图书预约**：对于已借出的图书，用户可以直接在线预约
- **图书收藏**：用户可以收藏喜欢的图书，方便后续查找
- **图书评价**：查看其他用户对图书的评价和评分

#### 📖 我的借阅

![我的借阅](./展示图/3.首页-我的借阅.png)

- **当前借阅**：查看所有正在借阅的图书，包括借阅日期、应还日期、续借次数等信息
- **一键续借**：对符合续借条件的图书进行一键续借操作
- **借阅历史**：查看历史借阅记录，包括借阅日期、归还日期、借阅状态等
- **逾期提醒**：醒目的逾期图书提醒，包括逾期天数和罚款金额
- **归还预约**：对于即将到期的图书，可以预约归还时间

#### 📅 我的预约

![我的预约](./展示图/4.首页-我的预约.png)

- **图书预约**：管理所有图书预约记录，包括预约状态、预约时间等
- **预约状态跟踪**：实时跟踪预约图书的状态，如待处理、已确认、已到馆等
- **预约取消**：用户可以在图书到馆前取消预约
- **到馆通知**：图书到馆后系统自动发送通知提醒用户
- **预约优先级**：根据用户类型和预约时间设置合理的预约优先级

#### 🪑 座位预约

![座位预约](./展示图/5.首页-座位预约.png)

- **可视化座位选择**：通过直观的座位图选择可用座位
- **阅览室实时状态**：显示各阅览室的当前占用情况和可用座位数量
- **多时段预约**：支持选择不同时间段进行座位预约
- **座位类型选择**：根据需要选择不同类型的座位（如安静区、讨论区等）
- **一键签到/签退**：通过扫码或手动操作完成座位的签到和签退
- **预约提醒**：预约成功后发送提醒，临近预约时间再次提醒

#### 🪑 我的座位

![我的座位](./展示图/6.首页-我的座位.png)

- **预约记录**：查看所有座位预约记录，包括预约时间、阅览室、座位号等
- **使用历史**：查看过去的座位使用记录
- **违规记录**：查看座位使用过程中的违规情况和处理结果
- **预约统计**：统计个人座位预约的使用情况和频率
- **偏好设置**：设置常用的阅览室和座位类型，方便快速预约

#### 👤 个人中心

![个人中心](./展示图/7.首页-个人中心.png)

- **个人信息管理**：查看和修改个人基本信息，如姓名、联系方式、证件信息等
- **密码修改**：安全的密码修改功能，支持密码强度验证
- **借阅权限**：查看个人的借阅权限，包括可借数量、借阅期限等
- **罚款记录**：查看所有罚款记录，包括罚款原因、金额、缴纳状态等
- **积分管理**：查看个人积分情况，了解积分获取和使用规则
- **证件管理**：管理个人证件信息，如学生证、教师证等

#### 📬 消息中心

![消息中心](./展示图/8.首页-消息中心.png)

- **系统通知**：接收图书馆发布的系统级通知和公告
- **借阅提醒**：借阅到期提醒、续借成功提醒、逾期提醒等
- **预约通知**：图书到馆通知、座位预约成功通知、预约取消通知等
- **咨询回复**：在线咨询的回复消息
- **消息分类**：按消息类型进行分类，方便查看和管理
- **消息已读标记**：标记已读消息，避免重复查看

#### 💬 咨询中心

![咨询中心](./展示图/9.首页-咨询中心.png)

- **在线咨询**：与图书馆工作人员进行实时在线沟通
- **常见问题FAQ**：浏览常见问题和解答，快速获取帮助
- **意见反馈**：提交对图书馆服务和系统的意见和建议
- **咨询历史**：查看历史咨询记录和回复
- **咨询分类**：按咨询类型进行分类，如图书借阅、座位预约、系统使用等
- **智能客服**：提供智能问答服务，快速解答常见问题

  1. 数据库准备: 创建数据库并配置连接信息

- **方式二: 传统部署**
  1. 构建前端: `npm run build`
  2. 构建后端: `mvn clean package`
  3. 部署前端: 将构建产物部署到Nginx
  4. 部署后端: 运行jar包或部署到应用服务器

| **前端优化**  | 路由懒加载、组件按需加载、静态资源压缩 | 使用Vue Router的懒加载、Element Plus的按需导入、Vite的构建优化 |
| **缓存策略**  | 合理使用浏览器缓存、服务端缓存      | 浏览器缓存策略、Redis缓存（可选）、本地缓存 |
| **异步处理**  | 定时任务异步执行、非阻塞IO       | 使用Spring Task、@Async注解、CompletableFuture |
| **分页查询**  | 大数据量分页处理            | 基于MyBatis-Plus的分页插件、滚动分页 |
| **响应式设计** | 前端响应式布局、后端自适应处理     | 使用Element Plus的响应式组件、后端自适应处理不同设备请求 |
| **负载均衡**  | 多实例部署、请求分发          | Nginx反向代理、负载均衡配置（可选） |
| **代码优化**  | 代码结构优化、算法优化         | 合理的代码结构设计、高效的算法实现 |

### 性能优化详细策略

**数据库层面**：
- 为频繁查询的字段创建索引，提高查询速度
- 使用MyBatis-Plus的性能分析插件，识别慢查询
- 配置HikariCP连接池，优化数据库连接管理
- 合理设计表结构，避免冗余字段和复杂关联
- 使用批量操作减少数据库交互次数

**前端层面**：
- 实现路由懒加载，减少初始加载时间
- 按需导入Element Plus组件，减少打包体积
- 使用Vite的构建优化，提高构建速度和运行性能
- 实现组件的条件渲染，减少DOM节点数量
- 使用虚拟列表处理大数据量展示

**后端层面**：
- 使用@Async注解实现异步处理，提高并发能力
- 实现定时任务的异步执行，避免阻塞主线程
- 使用CompletableFuture处理异步任务，提高代码可读性和执行效率
- 实现接口的缓存机制，减少重复计算和数据库查询
- 合理设计API接口，避免过度设计和不必要的计算

**部署层面**：
- 配置Nginx反向代理，实现负载均衡
- 合理配置服务器资源，根据实际负载调整
- 实现系统监控，及时发现和解决性能瓶颈
- 定期进行性能测试和优化，持续改进系统性能
