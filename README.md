## 📖 项目简介

城市图书馆管理系统是一个面向城市公共图书馆/高校图书馆的现代化、智能化图书管理平台。采用前后端分离架构,支持多角色(读者/管理员/超级管理员)协同工作,提供完整的图书管理、借阅管理、座位预约、用户管理等功能。

### 🎯 设计目标

| 目标       | 说明                              |
| -------- | ------------------------------- |
| **高可用性** | 核心业务7×24小时稳定运行,目标可用性 99.9%      |
| **高性能**  | 接口响应时间 P99 < 500ms,支持并发用户数 500+ |
| **安全性**  | 符合等级保护2.0要求,数据加密传输和存储           |
| **可扩展性** | 模块化设计,支持功能水平扩展和垂直扩展             |
| **易维护性** | 清晰的代码结构、完善的文档、自动化测试覆盖           |

### 📋 项目背景

随着数字化转型的推进，传统图书馆管理系统已无法满足现代图书馆的业务需求。本项目旨在打造一个功能全面、用户友好、技术先进的图书馆管理系统，解决传统系统存在的痛点：

- **传统系统功能单一**：无法满足现代图书馆多元化的服务需求
- **用户体验差**：界面老旧，操作繁琐
- **管理效率低**：人工操作多，自动化程度低
- **数据孤岛**：各系统之间数据不互通，难以进行统一管理和分析
- **安全性不足**：缺乏完善的权限控制和数据保护机制

本系统通过整合现代信息技术，为图书馆提供全方位的管理解决方案，提升图书馆的服务质量和管理效率。

- 采用JWT实现无状态认证，便于系统水平扩展

### 部署策略

本项目支持多种部署方式，根据不同的环境和需求选择合适的部署策略：

#### 开发环境部署
- **方式一: 使用启动器 (推荐)**
  - 图形界面版: `python launcher_gui.py`
  - 命令行版: `python launcher.py start`

- **方式二: 手动启动**
  1. 数据库准备: 创建数据库并配置连接信息
  2. 启动后端: `mvn spring-boot:run`
  3. 启动前端: `npm run dev`

#### 生产环境部署
- **方式一: 容器化部署**
  - 使用Docker和Docker Compose进行容器化部署
  - 提供完整的Dockerfile和docker-compose.yml配置

- **方式二: 传统部署**
  1. 构建前端: `npm run build`
  2. 构建后端: `mvn clean package`
  3. 部署前端: 将构建产物部署到Nginx
  4. 部署后端: 运行jar包或部署到应用服务器

### 启动步骤

-- 创建数据库

## 📊 性能优化

| 优化项       | 说明                  | 技术实现                |
| --------- | ------------------- | -------------------- |
| **数据库优化** | 索引优化、查询优化、连接池配置     | 合理设计索引、使用MyBatis-Plus的性能分析、配置HikariCP连接池 |
| **前端优化**  | 路由懒加载、组件按需加载、静态资源压缩 | 使用Vue Router的懒加载、Element Plus的按需导入、Vite的构建优化 |
| **缓存策略**  | 合理使用浏览器缓存、服务端缓存      | 浏览器缓存策略、Redis缓存（可选）、本地缓存 |
| **异步处理**  | 定时任务异步执行、非阻塞IO       | 使用Spring Task、@Async注解、CompletableFuture |
| **分页查询**  | 大数据量分页处理            | 基于MyBatis-Plus的分页插件、滚动分页 |
| **响应式设计** | 前端响应式布局、后端自适应处理     | 使用Element Plus的响应式组件、后端自适应处理不同设备请求 |
| **负载均衡**  | 多实例部署、请求分发          | Nginx反向代理、负载均衡配置（可选） |
| **代码优化**  | 代码结构优化、算法优化         | 合理的代码结构设计、高效的算法实现 |

### 性能优化详细策略

**数据库层面**：
- 为频繁查询的字段创建索引，提高查询速度
- 使用MyBatis-Plus的性能分析插件，识别慢查询
- 配置HikariCP连接池，优化数据库连接管理
- 合理设计表结构，避免冗余字段和复杂关联
- 使用批量操作减少数据库交互次数

**前端层面**：
- 实现路由懒加载，减少初始加载时间
- 按需导入Element Plus组件，减少打包体积
- 使用Vite的构建优化，提高构建速度和运行性能
- 实现组件的条件渲染，减少DOM节点数量
- 使用虚拟列表处理大数据量展示

**后端层面**：
- 使用@Async注解实现异步处理，提高并发能力
- 实现定时任务的异步执行，避免阻塞主线程
- 使用CompletableFuture处理异步任务，提高代码可读性和执行效率
- 实现接口的缓存机制，减少重复计算和数据库查询
- 合理设计API接口，避免过度设计和不必要的计算

**部署层面**：
- 配置Nginx反向代理，实现负载均衡
- 合理配置服务器资源，根据实际负载调整
- 实现系统监控，及时发现和解决性能瓶颈
- 定期进行性能测试和优化，持续改进系统性能
