### 2.2 分层说明

#### Controller层（表现层）
- **职责**：接收HTTP请求，参数校验，调用Service层处理业务，封装返回结果
- **注解使用**：`@RestController`、`@RequestMapping`、`@GetMapping`/`@PostMapping`等
- **权限控制**：通过`@PreAuthorize`注解实现方法级权限控制
- **统一返回格式**：大部分接口返回 `Map<String, Object>` 或 `ResponseEntity<T>`

#### Service层（业务逻辑层）
- **职责**：实现核心业务逻辑，事务管理，数据校验和转换
- **设计模式**：采用接口+实现类的方式
- **关键Service**：
  - UserService：用户注册、登录、信息管理
  - BookService：图书CRUD、搜索、分类
  - BorrowService：借阅、归还、续借业务
  - ReservationService：预约排队机制
  - FineRecordService：罚款计算与支付
  - SeatService：座位预约签到签退
  - NotificationService：通知消息管理
  - ExportService：多格式数据导出

#### Model层（数据模型层）
- **核心实体**：
  - UserAccount：用户账户（支持学生/教师/VIP/普通读者）
  - Book：图书信息（ISBN、作者、出版社、分类等）
  - BorrowRecord：借阅记录（借还状态、续借次数）
  - Reservation：图书预约记录
  - Seat/SeatReservation/ReadingRoom：座位与阅览室
  - FineRecord：罚款记录
  - Notification：系统通知
  - SystemConfig：系统配置
  - PurchaseRequest/AcquisitionTask：采购与征订

#### Repository/Mapper层（数据访问层）
- **技术选型**：MyBatis-Plus
- **基础Mapper**：继承 `BaseMapper<T>`，提供CRUD能力
- **自定义查询**：使用 LambdaQueryWrapper 构建动态查询条件
- **分页支持**：内置 IPage 分页插件

### 2.3 设计模式使用

| 设计模式 | 应用场景 | 示例 |
|---------|---------|------|
| **分层架构** | 整体架构 | Controller → Service → Repository |
| **依赖注入** | 组件间协作 | @Autowired 构造器注入 |
| **过滤器模式** | JWT认证 | JwtAuthenticationFilter extends OncePerRequestFilter |
| **策略模式** | 罚款计算 | FineType枚举 + 不同计算策略 |
| **工厂模式** | Token生成 | JwtTokenProvider.generateToken() |
| **单例模式** | 配置组件 | @Component 标注的Bean |
| **模板方法** | 定时任务基类 | @Scheduled 注解驱动 |
| **观察者模式** | 事件通知 | NotificationService.createNotification() |

### 3.1 认证模块 (AuthController)

**基础路径**: `/api/auth`

| HTTP方法 | URL路径 | 方法名 | 权限要求 | 说明 |
|---------|--------|--------|---------|------|
| GET | `/api/auth/captcha` | getCaptcha() | 无需认证 | 获取图形验证码（4位字母数字） |
| POST | `/api/auth/login` | loginUser() | 无需认证 | 用户登录（用户名+密码+验证码） |
| POST | `/api/auth/refresh` | refreshToken() | 无需认证 | 刷新访问令牌（Refresh Token） |
| POST | `/api/auth/logout` | logoutUser() | 无需认证 | 用户登出（清除Refresh Token） |
| GET | `/api/auth/user/info` | getUserInfo() | 需要认证 | 获取当前登录用户信息 |

#### 3.1.1 登录接口详情

**POST** `/api/auth/login`

**请求参数 (Request Body)**:
```json
{
  "username": "string",      // 用户名（必填）
  "password": "string"       // 密码（必填）
}
```

**成功响应示例**:
```json
{
  "code": 200,
  "message": "Login successful",
  "accessToken": "eyJhbGciOiJIUzUxMiJ9...",
  "refreshToken": "uuid-token-string",
  "tokenType": "Bearer",
  "user": {
    "id": 1,
    "username": "admin",
    "realName": "管理员",
    "role": "ROLE_ADMIN",
    "userType": "ADMIN",
    "status": 1,
    "email": "admin@example.com"
  },
  "identityId": "STU20240001",
  "identityType": "student"
}
```

**错误响应示例**:
```json
{
  "code": 400,
  "message": "用户名或密码不能为空"
}
// 或
{
  "code": 401,
  "message": "Login failed: Bad credentials"
}
```

#### 3.5.1 借阅图书接口

**POST** `/api/borrow/borrow?bookId={bookId}&bookBarcode={barcode}`

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| bookId | Long | 是 | 图书ID |
| bookBarcode | String | 是 | 图书条形码 |

**业务逻辑**:
```java
1. 验证当前用户已登录
2. 检查图书是否存在且未删除
3. 检查图书是否有可用副本（availableCopies > 0）
4. 检查用户是否已借阅该图书且未归还
5. 创建借阅记录：
   - 设置默认借阅期限30天
   - 初始续借次数为0
   - 状态设为"借阅中"(status=0)
6. 更新图书可用副本数减1
7. 返回借阅结果
```

**成功响应**:
```json
{
  "success": true,
  "message": "借阅成功",
  "data": {
    "id": 1,
    "userId": 1,
    "bookId": 10,
    "borrowDate": "2026-04-11T10:00:00",
    "dueDate": "2026-05-11T10:00:00",
    "renewCount": 0,
    "status": 0
  }
}
```

**错误响应**:
```json
{ "success": false, "message": "图书不存在" }
{ "success": false, "message": "图书无可用副本" }
{ "success": false, "message": "您已借阅该图书，尚未归还" }
```

## 4. 安全机制设计

### 4.1 JWT认证流程

```
┌──────────┐     1.输入用户名密码      ┌──────────┐
│          │ ──────────────────────▶ │          │
│   客户端  │                          │  后端服务  │
│          │ ◀────────────────────── │          │
│          │   2.返回AccessToken      │          │
│          │     + RefreshToken       │          │
│          │                          │          │
│          │  3.后续请求携带Token     │          │
│          │ ──────────────────────▶ │          │
│          │   Authorization: Bearer  │          │
│          │   xxxxxx.yyyyy.zzzzz    │          │
│          │                          │          │
│          │ ◀────────────────────── │          │
│          │   4.返回请求数据         │          │
└──────────┘                          └──────────┘
```

**完整认证流程**:

1. **登录阶段**
   - 用户提交用户名和密码
   - 服务端验证凭据（AuthenticationManager）
   - 验证成功后生成JWT Access Token（短期有效）
   - 同时生成 Refresh Token（长期有效），存储到数据库
   - 返回双Token给客户端

2. **请求阶段**
   - 客户端在请求头携带 `Authorization: Bearer <access_token>`
   - JwtAuthenticationFilter 拦截请求
   - 从Header提取Token并验证有效性
   - 从Token解析用户名，加载用户详情（UserDetails）
   - 设置SecurityContext认证信息
   - 放行到目标Controller处理

3. **Token刷新阶段**
   - 当Access Token过期时，使用Refresh Token换取新的Access Token
   - 验证Refresh Token是否过期或被撤销
   - 生成新的Access Token返回

4. **登出阶段**
   - 客户端发送Refresh Token到后端
   - 后端删除数据库中的Refresh Token记录
   - Token立即失效

### 4.2 Token生成与验证

**JwtTokenProvider 核心方法** ([JwtTokenProvider.java](file:///g:/CIFS_Java_web/Javaweb/BookSystem_Over/spring-boot-backend/src/main/java/com/example/demo/security/JwtTokenProvider.java)):

```java
@Component
public class JwtTokenProvider {

    @Value("${jwt.secret}")
    private String jwtSecret;              // JWT密钥（从配置读取）

    @Value("${jwt.expiration-ms}")
    private int jwtExpirationMs;           // Token过期时间（毫秒）

    // 生成签名密钥（HS512算法）
    private SecretKey getSigningKey() {
        return Keys.hmacShaKeyFor(jwtSecret.getBytes(StandardCharsets.UTF_8));
    }

    // 生成Token（基于Authentication对象）
    public String generateToken(Authentication authentication) {
        String username = authentication.getName();
        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + jwtExpirationMs);

        return Jwts.builder()
                .subject(username)                    // 设置主题（用户名）
                .issuedAt(now)                       // 签发时间
                .expiration(expiryDate)              // 过期时间
                .signWith(getSigningKey(), Jwts.SIG.HS512)  // HS512签名
                .compact();
    }

    // 根据用户名直接生成Token（用于刷新场景）
    public String generateTokenFromUsername(String username) { ... }

    // 从Token中解析用户名
    public String getUsernameFromToken(String token) {
        Claims claims = Jwts.parser()
                .verifyWith(getSigningKey())
                .build()
                .parseSignedClaims(token)
                .getPayload();
        return claims.getSubject();
    }

    // 验证Token有效性
    public boolean validateToken(String token) {
        try {
            Jwts.parser()
                .verifyWith(getSigningKey())
                .build()
                .parseSignedClaims(token);
            return true;
        } catch (JwtException | IllegalArgumentException ex) {
            return false;  // Token无效或过期
        }
    }
}
```

**Token结构**:
```
Header.Payload.Signature

Header:
{
  "alg": "HS512",
  "typ": "JWT"
}

Payload:
{
  "sub": "username",          // 用户名
  "iat": 1712812345,          // 签发时间
  "exp": 1712898745           // 过期时间
}

Signature:
HMACSHA512(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  secret_key
)
```

### 4.3 Spring Security配置

**SecurityConfig 配置详解** ([SecurityConfig.java](file:///g:/CIFS_Java_web/Javaweb/BookSystem_Over/spring-boot-backend/src/main/java/com/example/demo/config/SecurityConfig.java)):

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity  // 启用方法级安全注解（@PreAuthorize）
public class SecurityConfig {

    private final JwtAuthenticationFilter jwtAuthenticationFilter;

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            // 1. CORS跨域配置
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            
            // 2. 禁用CSRF（因为使用Token认证）
            .csrf(csrf -> csrf.disable())
            
            // 3. 无状态Session（不使用Session存储认证状态）
            .sessionManagement(session -> 
                session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            
            // 4. 请求授权配置
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers(HttpMethod.OPTIONS, "/**").permitAll()  // OPTIONS预检请求放行
                .requestMatchers("/**").permitAll()  // 当前配置：所有请求都允许
                // 注意：实际权限控制在Controller层的@PreAuthorize注解
                .anyRequest().authenticated()
            )
            
            // 5. 禁用X-Frame-Options（允许iframe嵌入）
            .headers(headers -> 
                headers.frameOptions(frameOptions -> frameOptions.disable()));

        // 6. 添加JWT过滤器（在UsernamePasswordAuthenticationFilter之前）
        http.addFilterBefore(jwtAuthenticationFilter, 
            UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();  // BCrypt加密算法
    }

    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration configuration = new CorsConfiguration();
        configuration.setAllowedOriginPatterns(Arrays.asList("*"));  // 允许所有来源
        configuration.setAllowedHeaders(Arrays.asList("*"));          // 允许所有请求头
        configuration.setAllowedMethods(Arrays.asList("*"));          // 允许所有HTTP方法
        configuration.setAllowCredentials(true);                      // 允许携带Cookie
        configuration.setMaxAge(3600L);                               // 预检请求缓存1小时
        
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", configuration);
        return source;
    }
}
```

**CORS配置说明**:
- 允许跨域访问（前后端分离架构必需）
- 支持Credentials（允许前端携带Cookie/Authorization头）
- 预检请求缓存1小时减少OPTIONS请求

```java
    }

    // 2. 检查可借副本
    if (book.getAvailableCopies() <= 0) {
        return error("图书无可用副本");
    }

    // 3. 检查重复借阅
    if (已有未归还记录) {
        return error("您已借阅该图书，尚未归还");
    }

    // 4. 创建借阅记录
    BorrowRecord record = new BorrowRecord();
    record.setUserId(userId);
    record.setBookId(bookId);
    record.setBorrowDate(new Date());
    record.setDueDate(计算30天后);  // 默认借阅期限
    record.setRenewCount(0);
    record.setStatus(0);  // 0=借阅中
    
    borrowRecordMapper.insert(record);

    // 5. 更新图书可用数
    book.setAvailableCopies(book.getAvailableCopies() - 1);
    book.setStatus("borrowed");
    bookMapper.updateById(book);

    return success("借阅成功", record);
}
```

### 7.1 逾期检查任务

**任务类**: [OverdueCheckTask](file:///g:/CIFS_Java_web/Javaweb/BookSystem_Over/spring-boot-backend/src/main/java/com/example/demo/task/OverdueCheckTask.java)

```

2. 逐条处理逾期记录：
   a. 更新借阅记录状态为"逾期"(status=2)
   b. 计算罚款金额：
      - 逾期天数 = (当前时间 - 应还日期) / 天
      - 罚款金额 = 0.50元 × 有效逾期天数
      - 有效逾期天数 = 逾期天数 - 3天宽限期
   c. 创建罚款记录（FineRecord）
      - type: OVERDUE
      - amount: 计算得出的金额
      - paidStatus: 0 (未支付)
   d. 发送逾期通知给用户
      - 标题："图书逾期通知"
      - 内容：包含书名、应还日期、逾期天数、当前罚款金额
      - 类型：TYPE_OVERDUE

3. 输出执行日志：
   - 处理记录数
   - 创建罚款数
   - 发送通知数
```

**关键代码片段**:

```java
@Scheduled(cron = "0 0 1 * * ?")  // 每天01:00执行
@Transactional  // 保证事务一致性
public void checkOverdueRecords() {
    Date now = new Date();
    
    // 查询逾期记录
    LambdaQueryWrapper<BorrowRecord> queryWrapper = new LambdaQueryWrapper<>();
    queryWrapper.eq(BorrowRecord::getStatus, STATUS_BORROWING)  // 借阅中
                .lt(BorrowRecord::getDueDate, now)              // 已过期
                .eq(BorrowRecord::getDeleted, 0);               // 未删除
    
    List<BorrowRecord> overdueRecords = borrowRecordMapper.selectList(queryWrapper);
    
    for (BorrowRecord record : overdueRecords) {
        // 1. 更新状态为逾期
        updateRecordStatusToOverdue(record.getId());
        
        // 2. 创建罚款记录
        long diffDays = (now.getTime() - record.getDueDate().getTime()) / (24*60*60*1000);
        BigDecimal fineAmount = FINE_PER_DAY.multiply(new BigDecimal(Math.max(0, diffDays - 3)));
        createFineRecord(record, fineAmount);
        
        // 3. 发送通知
        sendOverdueNotification(record);
    }
}
```
