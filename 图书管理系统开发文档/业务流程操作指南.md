## 3. 用户管理流程

### 3.1 用户注册流程

**流程步骤：**
```
访问注册页面 → 填写注册表单 → 表单验证 → 提交注册 → 后端验证 → 创建账户 → 注册成功跳转
```

#### 3.1.1 前端操作界面描述

**涉及组件：** `RegisterView.vue` 或 `Login.vue`（含注册功能）

**表单字段要求：**

| 字段名 | 字段标识 | 是否必填 | 验证规则 | 说明 |
|--------|----------|:--------:|----------|------|
| 用户名 | username | ✅ | 3-20字符，唯一 | 登录账号 |
| 密码 | password | ✅ | 符合强度规则 | 见下方密码规则 |
| 确认密码 | confirmPassword | ✅ | 与密码一致 | - |
| 真实姓名 | realName | ✅ | 2-50字符 | 显示名称 |
| 学号/教职工号 | studentId/facultyId | 条件必填 | 唯一性校验 | 根据用户类型 |
| 手机号 | phone | 推荐 | 11位手机号 | 用于通知 |
| 邮箱 | email | 推荐 | 邮箱格式 | 用于通知 |
| 用户类型 | userType | ✅ | 枚举值 | READER/LIBRARIAN |

#### 3.1.2 密码强度验证规则（从Service提取）

**源码位置：** [UserService.java](file:///g:/CIFS_Java_web/Javaweb/BookSystem_Over/spring-boot-backend/src/main/java/com/example/demo/service/UserService.java#L129-L133)

```java

    throw new RuntimeException(validationResult.getMessage());

```

**密码规则：**
- 最小长度：8位字符
- 必须包含：大写字母、小写字母、数字
- 可选包含：特殊字符
- 不能包含：连续3个相同字符、常见弱密码

#### 3.1.3 后端处理逻辑（registerUser方法）

**源码位置：** [UserService.java#L122-L144](file:///g:/CIFS_Java_web/Javaweb/BookSystem_Over/spring-boot-backend/src/main/java/com/example/demo/service/UserService.java#L122-L144)

**处理步骤：**

1. **用户名唯一性检查**
   ```java
   QueryWrapper<UserAccount> queryWrapper = new QueryWrapper<>();
   queryWrapper.eq("username", userAccount.getUsername());
   if (userAccountMapper.selectOne(queryWrapper) != null) {
       return false;  // 用户名已存在
   }
   ```

2. **密码强度验证**
   - 调用 `PasswordValidator.validate()` 进行强度检查
   - 不通过则抛出异常并提示具体原因

3. **密码加密存储**
   ```java
   userAccount.setPassword(passwordEncoder.encode(userAccount.getPassword()));
   ```
   - 使用BCrypt算法加密
   - 加密后存入数据库

4. **设置初始状态**
   ```java
   userAccount.setStatus(1);      // 状态：正常
   userAccount.setDeleted(0);     // 软删除标记：未删除
   userAccount.setCreateTime(new Date());
   userAccount.setUpdateTime(new Date());
   ```

5. **插入数据库**
   ```java
   userAccountMapper.insert(userAccount);
   ```

6. **返回结果**
   - 成功：返回 `true`
   - 失败：返回 `false` 或抛出异常

#### 3.1.4 可能的错误情况及提示

| 错误场景 | 错误信息 | 处理建议 |
|----------|----------|----------|
| 用户名为空 | "用户名不能为空" | 填写用户名 |
| 用户名已存在 | "用户名已存在" | 更换用户名 |
| 密码为空 | "密码不能为空" | 输入密码 |
| 密码强度不足 | "密码必须包含大小写字母和数字" | 按规则设置密码 |
| 学号重复 | "学号已存在" | 核实学号是否正确 |
| 教职工号重复 | "教职工号已存在" | 核实教职工号 |
| 数据库异常 | "注册失败，请稍后重试" | 联系管理员 |

#### 3.2.3 后端认证流程

**认证方式：** JWT (JSON Web Token)

**处理步骤：**

1. **获取验证码**
   ```javascript
   // 前端调用
   const res = await authApi.getCaptcha()
   // 返回：captchaImage（Base64图片）+ captchaId（验证码ID）
   ```

2. **提交登录请求**
   ```javascript
   await authStore.login({
     username: form.username,
     password: form.password,
     captcha: form.captcha,
     remember: form.remember
   })
   ```

3. **后端验证流程**
   - 验证码校验
   - 用户名查询（支持多字段匹配）
   - 密码比对（BCrypt）
   - 用户状态检查（status=1）
   - 生成JWT Token
   - 返回Token和用户信息

4. **Token存储**
   - localStorage：持久化存储
   - 自动携带：每次API请求的Authorization头

5. **登录成功跳转**
   ```javascript
   const redirect = (route.query.redirect as string) || '/'
   router.push(redirect)  // 跳转到首页或之前页面
   ```

**后端处理：** [UserService.updatePassword()](file:///g:/CIFS_Java_web/Javaweb/BookSystem_Over/spring-boot-backend/src/main/java/com/example/demo/service/UserService.java#L205-L223)

```java
public boolean updatePassword(String username, String newPassword) {
    // 1. 密码强度验证
    PasswordValidator.ValidationResult validationResult = PasswordValidator.validate(newPassword);
    if (!validationResult.isValid()) {
        throw new RuntimeException(validationResult.getMessage());
    }
    
    // 2. 查询用户
    QueryWrapper<UserAccount> queryWrapper = new QueryWrapper<>();
    queryWrapper.eq("username", username).eq("deleted", 0);
    UserAccount userAccount = userAccountMapper.selectOne(queryWrapper);
    
    // 3. 加密新密码并更新
    if (userAccount != null) {
        userAccount.setPassword(passwordEncoder.encode(newPassword));
        userAccount.setUpdateTime(new Date());
        userAccountMapper.updateById(userAccount);
        return true;
    }
    return false;
}
```

```java
public BookImportBatch processBatchImport(Long batchId, List<Book> books) {

    int failCount = 0;

    for (Book book : books) {
        try {
            book.setDeleted(0);
            book.setAvailableCopies(book.getTotalCopies());  // 初始全可借
            book.setStatus("available");
            book.setCreateTime(new Date());
            book.setUpdateTime(new Date());
            bookMapper.insert(book);

            failCount++;  // 单条失败不影响其他
        }
    }

    batch.setFailCount(failCount);
    batch.setStatus(1);  // 完成

}
```

```java
private BigDecimal calculateFine(Date dueDate, Date returnDate, BigDecimal fineRate) {
    if (returnDate.after(dueDate)) {
        long diffDays = (returnDate.getTime() - dueDate.getTime()) / (24 * 60 * 60 * 1000);
        return fineRate.multiply(new BigDecimal(diffDays));
    }
    return BigDecimal.ZERO;  // 未逾期，无罚款
}
```

#### 5.5.1 逾期罚款计算公式

**源码位置：** [FineRecordService.calculateOverdueFine()](file:///g:/CIFS_Java_web/Javaweb/BookSystem_Over/spring-boot-backend/src/main/java/com/example/demo/service/FineRecordService.java#L89-L107)

```java
// 常量定义
private static final BigDecimal FINE_PER_DAY = new BigDecimal("0.50");    // 每日罚款：0.50元
private static final int GRACE_PERIOD_DAYS = 3;                           // 宽限期：3天
private static final BigDecimal MAX_FINE_AMOUNT = new BigDecimal("100.00"); // 上限：100元

public BigDecimal calculateOverdueFine(Date dueDate, Date returnDate) {
    // 1. 计算逾期总天数
    long diffMs = returnDate.getTime() - dueDate.getTime();
    long diffDays = diffMs / (24 * 60 * 60 * 1000);

    // 2. 减去宽限期
    long effectiveDays = diffDays - GRACE_PERIOD_DAYS;
    if (effectiveDays <= 0) {
        return BigDecimal.ZERO;  // 宽限期内不罚款
    }

    // 3. 计算罚款金额
    BigDecimal fine = FINE_PER_DAY.multiply(new BigDecimal(effectiveDays));
    
    // 4. 应用上限
    fine = fine.min(MAX_FINE_AMOUNT);
    
    // 5. 四舍五入保留2位小数
    fine = fine.setScale(2, RoundingMode.HALF_UP);

    return fine;
}
```

**罚款计算规则总结：**

| 参数 | 值 | 说明 |
|------|-----|------|
| 日罚款率 | 0.50元/天 | 每逾期一天罚款金额 |
| 宽限期 | 3天 | 逾期3天内不罚款 |
| 罚款上限 | 100.00元 | 单次逾期最多罚100元 |
| 计算精度 | 2位小数 | 四舍五入 |

**计算示例：**

| 01-15 | 01-18 | 3天 | 0天 | 0元 | **0元**（宽限期内） |
| 01-15 | 01-20 | 5天 | 2天 | 1.00元 | **1.00元** |
| 01-15 | 02-01 | 17天 | 14天 | 7.00元 | **7.00元** |
| 01-15 | 03-15 | 59天 | 56天 | 28.00元 | **28.00元** |
| 01-15 | 05-15 | 90天 | 87天 | 43.50元 | **43.50元** |
| 01-15 | 08-15 | 182天 | 179天 | 89.50元 | **89.50元** |

```java
@Transactional
public boolean cancelReservation(Long reservationId, Long userId) {
    Reservation reservation = reservationMapper.selectById(reservationId);
    
    // 1. 校验记录存在性和权限
    if (reservation == null || reservation.getDeleted() == 1) {
        throw new RuntimeException("预约记录不存在");
    }
    if (!reservation.getUserId().equals(userId)) {
        throw new RuntimeException("无权取消此预约");
    }

    int result = reservationMapper.updateById(reservation);
    return result > 0;
}
```

**不可取消的状态：**
- `STATUS_CANCELLED` (2)：已取消
- `STATUS_COMPLETED` (3)：已完成（已借阅）

#### 7.2.1 签到时间窗口

**常量定义：**
```java
private static final int CHECKIN_BEFORE_MINUTES = 15;  // 可提前15分钟签到
private static final int CHECKIN_AFTER_MINUTES = 30;   // 开始后30分钟内签到
```

**签到时间规则：**

| 时间段 | 状态 | 说明 |
|--------|------|------|
| 开始时间前 > 15分钟 | ❌ 不可签到 | "签到时间过早，请在开始时间前15分钟内签到" |
| 开始时间前 0-15分钟 | ✅ 可以签到 | 正常签到 |
| 开始时间后 0-30分钟 | ✅ 可以签到 | 正常签到（可能迟到但不违约） |
| 开始时间后 > 30分钟 | ⚠️ 违约签到 | "签到超时，已标记为违约" |

**签到校验逻辑：**
```java
// 1. 只能在预约当天签到
Date today = clearTime(new Date());
Date reserveDate = clearTime(reservation.getReserveDate());
if (!today.equals(reserveDate)) {
    throw new RuntimeException("只能在预约当天签到");
}

// 2. 检查签到时间窗口
String currentTime = new SimpleDateFormat("HH:mm").format(new Date());
String startTime = reservation.getStartTime();

SimpleDateFormat sdf = new SimpleDateFormat("HH:mm");
Date current = sdf.parse(currentTime);
Date start = sdf.parse(startTime);

// 计算允许签到的时间范围
Calendar beforeStart = Calendar.getInstance();
beforeStart.setTime(start);
beforeStart.add(Calendar.MINUTE, -CHECKIN_BEFORE_MINUTES);  // 开始前15分钟

Calendar afterStart = Calendar.getInstance();
afterStart.setTime(start);
afterStart.add(Calendar.MINUTE, CHECKIN_AFTER_MINUTES);     // 开始后30分钟

// 3. 判断是否在时间窗口内
if (current.before(beforeStart.getTime())) {
    throw new RuntimeException("签到时间过早，请在开始时间前" + CHECKIN_BEFORE_MINUTES + "分钟内签到");
}

if (current.after(afterStart.getTime())) {
    // 超过30分钟，标记违约
    UserAccount user = userAccountMapper.selectById(userId);
    if (user != null) {
        int violationCount = user.getViolationCount() != null ? user.getViolationCount() : 0;
        user.setViolationCount(violationCount + 1);       // 违约次数+1
        user.setLastViolationTime(new Date());             // 记录最后违约时间
        userAccountMapper.updateById(user);
    }
    throw new RuntimeException("签到超时，已标记为违约");
}
```

### 7.3 座位签退流程

**流程步骤：**

**涉及方法：** [SeatService.checkOut()](file:///g:/CIFS_Java_web/Javaweb/BookSystem_Over/spring-boot-backend/src/main/java/com/example/demo/service/SeatService.java#L386-L413)

```java
@Transactional
public boolean checkOut(Long reservationId, Long userId) {
    SeatReservation reservation = seatReservationMapper.selectById(reservationId);
    
    // 1. 校验记录和权限
    if (reservation == null || reservation.getDeleted() == 1) {
        throw new RuntimeException("预约记录不存在");
    }
    if (!reservation.getUserId().equals(userId)) {
        throw new RuntimeException("无权操作此预约");
    }
    
    // 2. 只有已签到状态才能签退
    if (reservation.getStatus() != STATUS_CHECKED_IN) {
        throw new RuntimeException("该预约无法签退");
    }

    // 3. 更新预约状态
    reservation.setStatus(STATUS_CHECKED_OUT);  // 2=已签退
    reservation.setCheckOutTime(new Date());     // 记录签退时间
    int result = seatReservationMapper.updateById(reservation);

    if (result > 0) {
        // 4. 释放座位
        Seat seat = seatMapper.selectById(reservation.getSeatId());
        if (seat != null) {
            seat.setStatus(SEAT_AVAILABLE);  // 0=空闲
            seatMapper.updateById(seat);
        }
    }

    return result > 0;
}
```

**签退说明：**
- 签退后座位立即释放，可供他人预约
- 签退时间会被记录用于统计分析
- 提前签退不产生违约

### 7.4 违规处理机制

#### 7.4.1 违规行为认定

| 违规行为 | 触发条件 | 处理方式 |
|----------|----------|----------|
| **未按时签到** | 超过开始时间30分钟后签到 | 违约次数+1 |
| **未签到违约** | 预约当天未签到（定时任务扫描） | 违约次数+1 |
| **多次违规** | 累计违约≥3次 | 禁止预约7天 |

#### 7.4.2 违约次数累计规则

```java
// 违约次数累加（不重置）
int violationCount = user.getViolationCount() != null ? user.getViolationCount() : 0;
user.setViolationCount(violationCount + 1);
user.setLastViolationTime(new Date());  // 更新最后违约时间
userAccountMapper.updateById(user);
```

**重要说明：**
- 违约次数是**永久累计**的（不清零）
- 但禁止期只看最近一次违约时间
- 禁止期满后如果违约次数仍≥3次，下次违约还会禁止7天

#### 7.4.3 禁止预约期限计算

```
禁止结束时间 = 最后一次违约时间 + 7天
```

**示例：**
```
第1次违约：01-01 → 违约次数=1 → 可正常预约
第2次违约：01-05 → 违约次数=2 → 可正常预约
第3次违约：01-10 → 违约次数=3 → 禁止至 01-17
01-18 尝试预约 → 禁止期已过 → 可正常预约（但违约次数仍为3）
第4次违约：01-20 → 违约次数=4 → 再次禁止至 01-27
...以此类推
```

```java
@Transactional
public PurchaseRequest reviewPurchaseRequest(Long requestId, Long reviewerId, Integer status, String reviewComment) {
    PurchaseRequest request = purchaseRequestMapper.selectById(requestId);
    
    // 1. 校验申请存在性
    if (request == null || request.getDeleted() == 1) {
        throw new RuntimeException("采购申请不存在");
    }

    // 2. 检查是否已审核
    if (request.getStatus() != STATUS_PENDING) {
        throw new RuntimeException("该采购申请已被审核");
    }

    // 3. 更新审核结果
    request.setReviewerId(reviewerId);     // 审核人ID
    request.setStatus(status);             // 1=通过 2=驳回
    request.setReviewComment(reviewComment); // 审核意见
    request.setUpdateTime(new Date());
    
    purchaseRequestMapper.updateById(request);
    return request;
}
```

```java
    return notificationMapper.update(null, updateWrapper);  // 返回更新的条数
```

#### 12.8.4 删除通知

**涉及方法：** [NotificationService.deleteNotification()](file:///g:/CIFS_Java_web/Javaweb/BookSystem_Over/spring-boot-backend/src/main/java/com/example/demo/service/NotificationService.java#166-L175)

```java
public boolean deleteNotification(Long notificationId, Long userId) {
    Notification notification = notificationMapper.selectById(notificationId);
    if (notification == null || !notification.getUserId().equals(userId)) {
        return false;
    }
    LambdaUpdateWrapper<Notification> updateWrapper = new LambdaUpdateWrapper<>();
    updateWrapper.eq(Notification::getId, notificationId)
            .set(Notification::getDeleted, 1);  // 软删除
    return notificationMapper.update(null, updateWrapper) > 0;
}
```
