## 2. 前端架构概述

### 2.1 技术选型说明

本系统采用现代化的前端技术栈，具体选型及理由如下：

#### 核心技术栈
- **Vue 3 + Composition API**: 采用Vue 3的最新Composition API（`<script setup>`语法），提供更好的TypeScript支持和逻辑复用能力
- **TypeScript**: 全项目使用TypeScript进行类型约束，提升代码质量和开发体验
- **Vite**: 新一代前端构建工具，提供极速的开发服务器和优化的生产构建
- **Element Plus**: 基于Vue 3的企业级UI组件库，提供丰富的组件和完善的文档
- **Pinia**: Vue官方推荐的状态管理库，支持TypeScript，API简洁直观
- **vue-i18n**: 国际化解决方案，支持多语言切换
- **Axios**: HTTP客户端，用于与后端API通信

#### 辅助工具
- **ECharts**: 数据可视化图表库，用于统计报表展示
- **xlsx**: Excel文件处理库，用于数据导入导出功能
- **@element-plus/icons-vue**: Element Plus图标库
- **NProgress**: 页面加载进度条

### 2.3 设计理念

#### 组件化设计
- 采用Vue 3的Composition API，将逻辑按功能点组织
- 每个页面/功能模块独立封装为.vue单文件组件
- 公共组件抽取到`components`目录，提高复用性
- 使用TypeScript接口定义Props和Emits类型

#### 响应式设计
- 使用Element Plus的响应式栅格系统（el-row/el-col）
- 支持多种屏幕尺寸适配：xs(<768px)、sm(≥768px)、md(≥992px)、lg(≥1200px)、xl(≥1920px)
- CSS变量系统实现主题切换（支持暗色模式）
- 移动端友好的交互设计

#### 渐进式增强
- 路由懒加载：所有页面组件采用动态import()实现按需加载
- 权限控制：路由守卫实现细粒度的访问控制
- 错误处理：统一的错误处理机制和用户提示
- 性能优化：虚拟滚动、分页加载、图片懒加载等

## 4. 路由设计（重点章节）

### 4.1 路由配置总览

路由配置文件位置：[router/index.ts](src/router/index.ts)

本项目采用嵌套路由结构，分为三大路由区域：
1. **公开路由** - 登录页、404页面
2. **用户端路由** - 使用DefaultLayout布局，包含所有用户功能页面
3. **管理员路由** - 使用AdminLayout布局，包含所有管理功能页面

### 4.2 完整路由表

#### 4.2.1 公开页面路由

| 路径(path) | 路由名称(name) | 组件(Component) | 页面标题(meta.title) | 需要认证(requiresAuth) | 所需角色 | 说明 |
|------------|---------------|-----------------|---------------------|----------------------|---------|------|
| `/login` | `Login` | `LoginView.vue` | 登录 | `false` | 无 | 登录页面，支持验证码、记住登录 |
| `/404` | `NotFound` | `NotFoundView.vue` | 页面不存在 | - | - | 404错误页面 |
| `/:pathMatch(.*)*` | - | redirect → `/404` | - | - | - | 通配符重定向到404 |

### 4.4 路由守卫机制

路由守卫配置文件：[router/guard.ts](src/router/guard.ts)

```typescript
// 路由守卫核心逻辑
router.beforeEach(async (to, _from, next) => {
  // 1. 设置页面标题
  document.title = to.meta.title ? `${to.meta.title} - 城市图书馆管理系统` : '城市图书馆管理系统'

  // 2. 加载用户信息（如果已登录但用户信息为空）
  if (authStore.isLoggedIn && !authStore.user) {
    await authStore.initAuth()
  }

  // 3. 已登录用户访问逻辑
  if (authStore.isLoggedIn) {
    if (to.path === '/login') {
      next({ path: (to.query.redirect as string) || '/' })  // 已登录跳转首页
      return
    }
    if (to.meta.requiresAdmin) {
      if (!authStore.isAdmin) {
        next({ path: '/' })  // 非管理员重定向到首页
        return
      }
    }
    next()
    return
  }

  // 4. 未登录用户访问受保护页面
  if (to.meta.requiresAuth !== false) {
    next({ path: '/login', query: { redirect: to.fullPath } })  // 重定向到登录页并保存原路径
    return
  }
  next()
})
```

**权限控制流程：**
1. 访问任何页面时自动设置浏览器标题
2. 如果已登录但用户信息缺失，自动获取用户信息
3. 已登录用户访问登录页时，自动重定向到首页或之前要访问的页面
4. 已登录非管理员用户访问管理员页面时，重定向到首页
5. 未登录用户访问需要认证的页面时，重定向到登录页并记录原始路径

### 6.3 组件通信方式

#### Props/Emits（父子组件通信）
这是最常用的通信方式：

**父传子（Props）：**
```vue
<!-- 父组件 -->
<BookCard :book="book" :mode="'grid'" @borrow="handleBorrow" />

<!-- 子组件接收 -->
<script setup lang="ts">
interface Props { book: Book; mode?: 'grid' | 'list' }
const props = withDefaults(defineProps<Props>(), { mode: 'grid' })
</script>
```

**子传父（Emits）：**
```vue
<!-- 子组件触发 -->
defineEmits<{ borrow: [book: Book]; click: [book: Book] }>()

// 触发事件
emit('borrow', book)

<!-- 父组件监听 -->
<BookCard @borrow="handleBorrow" @click="handleClick" />
```

#### Provide/Inject（跨层级通信）
用于深层组件传递数据，如布局组件向深层子组件传递全局状态：

```typescript
// 父组件（Layout）
provide('authStore', authStore)
provide('isCollapsed', isCollapsed)

// 子组件
const authStore = inject('authStore')
const isCollapsed = inject('isCollapsed')
```

#### Store（全局状态管理）
通过Pinia Store实现跨组件状态共享：

```typescript
// 在任意组件中使用
const bookStore = useBookStore()
const authStore = useAuthStore()

// 读取状态
console.log(bookStore.list)

// 调用方法
await bookStore.fetchBooks(params)
```

#### Router（路由通信）
通过路由参数传递数据：

```typescript
// 路由定义
{ path: '/books/:id', name: 'BookDetail', component: BookDetailView }

// 获取参数
const route = useRoute()
const bookId = Number(route.params.id)

// 编程式导航

```

### 7.2 请求拦截器

```typescript
instance.interceptors.request.use(
  (config: InternalAxiosRequestConfig) => {
    // 1. Token注入 - 从localStorage或sessionStorage获取token
    const token = localStorage.getItem('token') || sessionStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }

    // 2. Accept-Language头设置 - 发送当前语言设置
    config.headers['Accept-Language'] = localStorage.getItem('locale') || 'zh-CN'

    // 3. FormData特殊处理 - 上传文件时移除Content-Type让浏览器自动设置
    if (config.data instanceof FormData) {
      delete config.headers['Content-Type']
    }

    return config
  },
  (error) => Promise.reject(error)
)
```

**请求头说明：**

| Header | 值来源 | 用途 |
|--------|-------|------|
| `Authorization` | `Bearer {token}` | 身份认证令牌 |
| `Accept-Language` | localStorage.locale | 服务端返回对应语言的响应 |
| `Content-Type` | `application/json` 或 自动 | 请求体格式 |

### 7.3 响应拦截器

```typescript
instance.interceptors.response.use(
  (response: AxiosResponse) => {
    const res = response.data

    // 1. 业务错误码检查 - code不为0或200视为业务错误
    if (res.code !== undefined && res.code !== 0 && res.code !== 200) {
      ElMessage.error(res.message || '请求失败')

      // 2. Token过期处理 - 401状态清除认证信息并跳转登录
      if (res.code === 401) {
        localStorage.removeItem('token')
        sessionStorage.removeItem('token')
        window.location.href = `/login?redirect=${window.location.pathname}`
      }

      return Promise.reject(new Error(res.message || '请求失败'))
    }

    return response  // 成功则返回完整response
  },
  (error) => {
    // 3. HTTP错误状态处理
    if (error.response?.status === 403) {
      ElMessage.error('无权限访问，请重新登录')
    } else if (error.response?.status === 401) {
      localStorage.removeItem('token')
      sessionStorage.removeItem('token')
      window.location.href = '/login'
    } else {
      const message = error.response?.data?.message || error.message || '网络错误'
      ElMessage.error(message)
    }

    return Promise.reject(error)
  }
)
```

**错误处理策略：**

| HTTP状态码 | 处理方式 | 用户提示 |
|-----------|---------|---------|
| 200 + code=401 | 清除Token，跳转登录页 | （自动处理） |
| 200 + code=其他错误 | 显示错误消息 | `res.message` |
| HTTP 401 | 清除Token，跳转登录页 | 请先登录 |
| HTTP 403 | 显示错误提示 | 无权限访问 |
| 其他HTTP错误 | 显示错误消息 | 具体错误信息或"网络错误" |

### 8.2 状态结构详解

#### authStore.ts - 认证状态Store
**文件位置:** [stores/authStore.ts](src/stores/authStore.ts)

```typescript
export const useAuthStore = defineStore('auth', () => {
  // ===== State =====
  const token = ref<string>(
    localStorage.getItem('token') || sessionStorage.getItem('token') || ''
  )
  const user = ref<any>(null)

  // ===== Getters =====
  const isLoggedIn = computed(() => !!token.value)
  const isAdmin = computed(() => {
    if (!user.value) return false
    const type = (user.value.userType || '').toUpperCase()
    const role = (user.value.role || '').toUpperCase()
    return type === 'ADMIN' || role === 'ADMIN' || role === 'ROLE_ADMIN'
  })

  // ===== Actions =====
  
  /** 初始化认证状态 */
  async function initAuth() {
    const savedToken = localStorage.getItem('token') || sessionStorage.getItem('token')
    if (savedToken) {
      token.value = savedToken
      if (!user.value) {
        await getUserInfo()
      }
    }
  }

  /** 用户登录 */
  async function login(loginData: { 
    username: string; password: string; 
    captcha: string; remember?: boolean 
  }) {
    const res: any = await authApi.login(loginData)
    const accessToken = res.accessToken || res.token
    token.value = accessToken
    user.value = res.user
    
    // 根据记住我选项决定存储位置
    if (loginData.remember) {
      localStorage.setItem('token', accessToken)
      localStorage.setItem('user', JSON.stringify(res.user))
      sessionStorage.removeItem('token')
      sessionStorage.removeItem('user')
    } else {
      sessionStorage.setItem('token', accessToken)
      sessionStorage.setItem('user', JSON.stringify(res.user))
      localStorage.removeItem('token')
      localStorage.removeItem('user')
    }
  }

  /** 获取用户信息 */
  async function getUserInfo() {
    const res: any = await authApi.getUserInfo()
    user.value = res.data || res
  }

  /** 用户登出 */
  function logout() {
    token.value = ''
    user.value = null
    localStorage.removeItem('token')
    localStorage.removeItem('user')
    sessionStorage.removeItem('token')
    sessionStorage.removeItem('user')
    authApi.logout().catch(() => {})
  }

  /** 从存储加载用户信息 */
  function loadUserFromStorage() {
    if (!user.value) {
      const savedUser = localStorage.getItem('user') || sessionStorage.getItem('user')
      if (savedUser) {
        user.value = JSON.parse(savedUser)
      }
    }
  }

  loadUserFromStorage()

  return { token, user, isLoggedIn, isAdmin, initAuth, login, getUserInfo, logout }
})
```

| `user` | `Ref<any>` | 当前登录用户信息 | localStorage/sessionStorage |
| `isLoggedIn` | `ComputedRef<boolean>` | 是否已登录（派生状态） | - |
| `isAdmin` | `ComputedRef<boolean>` | 是否是管理员（派生状态） | - |

### 8.3 持久化存储

本项目使用浏览器Web Storage进行状态持久化：

#### 存储策略

| 存储键(Key) | 存储类型 | 存储内容 | Store | 说明 |
|-------------|---------|---------|-------|------|
| `token` | localStorage/sessionStorage | JWT认证令牌 | authStore | 根据"记住登录"选择存储位置 |
| `user` | localStorage/sessionStorage | 用户信息JSON | authStore | 同上 |
| `locale` | localStorage | 当前语言设置 | i18n | 'zh-CN' 或 'en-US' |

#### 存储规则

**localStorage（持久存储）：**
- 用户勾选"记住登录"时使用
- 关闭浏览器后仍然保留
- 适用于长期保持登录状态的场景

**sessionStorage（会话存储）：**
- 用户未勾选"记住登录"时使用
- 关闭浏览器标签页/窗口后清除
- 适用于临时登录场景

**清理时机：**
- 用户主动登出时同时清除两个存储
- Token过期（401响应）时清除
- 路由守卫检测到无效Token时清除

export function loadLocaleMessages(locale: string) {

/** 切换语言（便捷方法） */
export function toggleLocale() {
  const currentLocale = i18n.global.locale.value

  loadLocaleMessages(newLocale)
}

## 附录C：设计原则与最佳实践

### C1. 代码风格规范

1. **组件命名：** PascalCase（如`BookCard.vue`、`AdminDashboardView.vue`）
2. **文件命名：** camelCase（如`authStore.ts`、`bookApi.ts`）
3. **API函数命名：** 动词开头（如`getBooks`、`createBook`、`deleteUser`）
4. **Store命名：** use + 名词 + Store（如`useAuthStore`、`useBookStore`）
5. **常量命名：** UPPER_SNAKE_CASE（如`MAX_BORROW_LIMIT`）

### C2. 错误处理规范

```typescript
// 正确的错误处理模式
try {
  const res = await someApi.call()
  // 处理成功结果
} catch (error: any) {
  // 1. 记录错误日志（开发环境）
  console.error('Operation failed:', error)
  
  // 2. 显示用户友好的错误提示
  ElMessage.error(error?.message || '操作失败，请稍后重试')
  
  // 3. 可选：恢复UI状态
  loading.value = false
}
```

### C3. 性能优化建议

1. **路由懒加载：** 所有页面组件使用动态import()实现按需加载
2. **列表虚拟滚动：** 大数据量列表考虑使用虚拟滚动
3. **图片懒加载：** 使用loading="lazy"或Intersection Observer
4. **防抖节流：** 搜索输入、窗口resize等高频操作使用防抖
5. **缓存策略：** 合理利用Store缓存API响应数据
6. **代码分割：** 第三方库按需引入，避免打包过大
