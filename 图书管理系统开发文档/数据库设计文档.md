## 2. 数据库概述

### 2.1 数据库选型说明（MySQL）

本系统选用MySQL作为关系型数据库管理系统，主要基于以下考虑：

- **开源免费**：降低系统部署成本
- **成熟稳定**：经过大规模生产环境验证
- **高性能**：支持InnoDB存储引擎，支持事务、行级锁
- **丰富的数据类型**：支持TEXT、JSON等复杂数据类型
- **完善的索引机制**：支持B-Tree、Hash等多种索引类型
- **良好的中文支持**：utf8mb4字符集完美支持Unicode，包括emoji表情

### 2.2 设计原则

#### 范式规范
- **第一范式（1NF）**：所有字段都是原子性的，不可再分
- **第二范式（2NF）**：非主键字段完全依赖于主键
- **第三范式（3NF）**：非主键字段不传递依赖于主键
- **适度反范式**：在book表中冗余category_name字段，避免频繁JOIN查询

#### 命名规范
- **表名**：使用小写字母和下划线，采用名词单数形式（如user_account、borrow_record）
- **字段名**：使用小写字母和下划线（如user_id、create_time）
- **主键**：统一使用`id`字段，BIGINT自增类型
- **外键**：采用`关联表_id`格式（如user_id、book_id）
- **时间字段**：统一使用`_time`后缀（如create_time、update_time）
- **状态字段**：统一使用`status`或`deleted`
- **布尔标识**：使用`has_`或`is_`前缀（如has_power、is_read）

#### 通用字段规范
所有业务表均包含以下标准字段：
- `id` - 主键ID（BIGINT AUTO_INCREMENT）
- `deleted` - 逻辑删除标记（TINYINT, 0-未删除, 1-已删除）
- `create_time` - 创建时间（DATETIME, DEFAULT CURRENT_TIMESTAMP）
- `update_time` - 更新时间（DATETIME, ON UPDATE CURRENT_TIMESTAMP）

### 2.3 字符集和排序规则

```sql
CHARACTER SET utf8mb4          -- 支持4字节UTF-8编码，包括所有Unicode字符
COLLATE utf8mb4_unicode_ci     -- 不区分大小写的Unicode排序规则
```

**选择理由**：
- utf8mb4相比utf8支持完整的Unicode字符集，包括emoji表情符号
- unicode_ci排序规则支持多语言环境下的正确排序和比较

### 2.4 存储引擎选择

统一使用**InnoDB**存储引擎：

```sql
ENGINE=InnoDB
```

**InnoDB优势**：
- 支持事务（ACID特性）
- 支持行级锁，提高并发性能
- 支持外键约束，保证数据完整性
- 支持崩溃恢复（crash-safe）
- 支持MVCC（多版本并发控制）

2. **用户-图书评价关系**
   - 中间表：book_review
   - 用户 ⇄ 图书（通过bookId和userId联合唯一索引）

3. **用户-图书推荐关系**
   - 中间表：book_recommendation
   - 用户 ⇄ 图书

4. **用户-图书借阅关系**
   - 中间表：borrow_record
   - 用户 ⇄ 图书（同一用户可多次借阅同一本书）

5. **用户-图书预约关系**
   - 中间表：reservation
   - 用户 ⇄ 图书

## 4. 数据表详细设计

### 4.1 user_account（用户账户表）

**表说明**：存储系统所有用户的账户信息，包括读者、学生、教师、管理员等。

|--------|----------|----------|--------|------|
| id | BIGINT | NOT NULL | AUTO_INCREMENT | 主键ID |
| username | VARCHAR(50) | NOT NULL | - | 用户名（唯一） |
| password | VARCHAR(100) | NOT NULL | - | 密码（BCrypt加密） |
| user_type | VARCHAR(20) | NOT NULL | 'READER' | 用户类型：READER/STUDENT/TEACHER/INTERNATIONAL |
| real_name | VARCHAR(50) | 可空 | NULL | 真实姓名 |
| phone | VARCHAR(20) | 可空 | NULL | 联系电话 |
| email | VARCHAR(100) | 可空 | NULL | 电子邮箱 |
| id_card | VARCHAR(30) | 可空 | NULL | 身份证件号 |
| student_id | VARCHAR(20) | 可空 | NULL | 学生学号（唯一） |
| faculty_id | VARCHAR(20) | 可空 | NULL | 教职工号（唯一） |
| user_code | VARCHAR(20) | 可空 | NULL | 用户编号（唯一） |
| campus | VARCHAR(100) | 可空 | NULL | 校区 |
| college | VARCHAR(100) | 可空 | NULL | 院系 |
| grade | VARCHAR(20) | 可空 | NULL | 年级 |
| class_name | VARCHAR(50) | 可空 | NULL | 班级 |
| counselor | VARCHAR(50) | 可空 | NULL | 辅导员 |
| institution | VARCHAR(100) | 可空 | NULL | 所属机构 |
| role | VARCHAR(20) | NOT NULL | 'ROLE_READER' | 系统角色 |
| language | VARCHAR(10) | NOT NULL | 'zh-CN' | 界面语言 |
| status | TINYINT | NOT NULL | 1 | 状态：0-禁用，1-正常 |
| deleted | TINYINT | NOT NULL | 0 | 逻辑删除：0-未删除，1-已删除 |

| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**附加说明**：
- **主键**：id（自增主键）
- **唯一索引**：
  - uk_username：username（保证用户名唯一）
  - uk_student_id：student_id（保证学号唯一）
  - uk_faculty_id：faculty_id（保证教职工号唯一）
  - uk_user_code：user_code（保证用户编号唯一）
- **普通索引**：
  - idx_campus：campus（按校区查询）
  - idx_college：college（按院系查询）
  - idx_grade：grade（按年级查询）
- **业务约束**：
  - 用户名必填且全局唯一
  - 密码使用BCrypt算法加密存储
  - student_id、faculty_id、user_code互斥，根据user_type决定哪个有值
  - 支持逻辑删除，不物理删除数据

### 4.3 category（图书分类表）

**表说明**：存储图书分类信息，支持层级结构（树形结构）。

| 字段名 | 数据类型 | 是否可空 | 默认值 | 说明 |
|--------|----------|----------|--------|------|
| id | BIGINT | NOT NULL | AUTO_INCREMENT | 主键ID |
| name | VARCHAR(50) | NOT NULL | - | 分类名称 |
| code | VARCHAR(20) | NOT NULL | - | 分类编码（唯一） |
| parent_id | BIGINT | 可空 | NULL | 父级分类ID（NULL表示顶级分类） |
| sort_order | INT | NOT NULL | 0 | 排序值 |
| description | VARCHAR(200) | 可空 | NULL | 分类描述 |
| deleted | TINYINT | NOT NULL | 0 | 逻辑删除：0-未删除，1-已删除 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**附加说明**：
- **主键**：id（自增主键）
- **唯一索引**：uk_code：code（保证分类编码唯一）
- **普通索引**：idx_parent_id：parent_id（快速查找子分类）
- **业务约束**：
  - 采用邻接表模型实现树形结构
  - code字段遵循《中国图书馆分类法》（A-Z共22个大类）
  - parent_id为NULL时表示顶级分类
  - sort_order用于同级分类的排序显示
  - 初始化数据包含22个一级分类（A-Z）

|--------|----------|----------|--------|------|

| user_id | BIGINT | NOT NULL | - | 用户ID（外键→user_account.id） |
| book_id | BIGINT | NOT NULL | - | 图书ID（外键→book.id） |
| book_barcode | VARCHAR(50) | 可空 | NULL | 图书条码 |
| borrow_date | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 借阅日期 |
| due_date | DATETIME | NOT NULL | - | 应还日期 |
| return_date | DATETIME | 可空 | NULL | 实际归还日期 |
| renew_count | INT | NOT NULL | 0 | 续借次数 |
| status | TINYINT | NOT NULL | 0 | 状态：0-借阅中，1-已归还，2-逾期，3-丢失 |
| operator_id | BIGINT | 可空 | NULL | 操作员ID |
| remarks | VARCHAR(500) | 可空 | NULL | 备注 |
| deleted | TINYINT | NOT NULL | 0 | 逻辑删除：0-未删除，1-已删除 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**附加说明**：
- **主键**：id（自增主键）

  - fk_borrow_user：user_id → user_account.id（ON DELETE RESTRICT）

  - idx_user_id：user_id（按用户查询借阅历史）

  - idx_borrow_date：borrow_date（按日期范围查询）
- **业务约束**：
  - 借阅日期默认为当前时间
  - 续借次数受系统配置renew_limit限制（默认3次）
  - status=0表示正在借阅，status=2表示逾期未还
  - 归还后return_date和status会更新
  - 不能删除有借阅记录的用户或图书（RESTRICT策略）

- **主键**：id（自增主键）

  - fk_fine_user：user_id → user_account.id（ON DELETE RESTRICT）

### 6.2 外键策略说明

#### ON DELETE 策略

- **RESTRICT（限制删除）**：如果子表存在关联记录，则禁止删除父表记录。适用于核心业务关系（如借阅记录、用户账户等），确保数据完整性。
  - 应用表：borrow_record、reservation、fine_record、inquiry、seat、seat_reservation（部分）、purchase_request、export_task、book_import_batch

- **CASCADE（级联删除）**：删除父表记录时，自动删除所有关联的子表记录。适用于从属关系明确的数据（如通知、评价、收藏、行为等）。
  - 应用表：notification、seat_reservation（room_id）、book_import_detail、book_review、user_favorite、user_behavior、book_recommendation

- **SET NULL（设为空）**：删除父表记录时，将子表的外键字段设为NULL。适用于允许孤立存在的记录（如罚款记录、导入明细、行为记录）。
  - 应用表：fine_record（borrow_record_id）、book_import_detail（book_id）、user_behavior（book_id）

#### ON UPDATE 策略

- 统一使用**RESTRICT**策略：禁止更新父表的主键值（因为主键是自增的，一般不会更新）。
