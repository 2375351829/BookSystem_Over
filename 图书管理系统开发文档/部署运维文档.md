## 4. 安装部署步骤

### 4.1 环境准备

#### 4.1.1 JDK 17 安装

##### Windows环境

**下载地址**:

- Oracle JDK: <https://www.oracle.com/java/technologies/downloads/>
- Adoptium Eclipse Temurin (推荐): <https://adoptium.net/>

**安装步骤**:

1. 下载JDK 17安装包（如 `jdk-17_windows-x64_bin.exe`）
2. 运行安装程序，选择安装路径（建议 `C:\Program Files\Java\jdk-17`）
3. 配置环境变量：

```powershell
# 设置JAVA_HOME
[System.Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Java\jdk-17", "User")

# 添加到PATH
$path = [System.Environment]::GetEnvironmentVariable("Path", "User")
[System.Environment]::SetEnvironmentVariable("Path", "$path;%JAVA_HOME%\bin", "User")
```

1. **验证安装**:

```bash
java -version
# 输出示例:
# java version "17.0.9" 2023-10-17 LTS
# Java(TM) SE Runtime Environment (build 17.0.9+11-LTS-251)
# Java HotSpot(TM) 64-Bit Server VM (build 17.0.9+11-LTS-251, mixed mode, sharing)

javac -version
# 输出: javac 17.0.9
```

##### Linux环境 (CentOS/Ubuntu)

```bash
# CentOS/RHEL
sudo yum install -y java-17-openjdk-devel

# Ubuntu/Debian
sudo apt update
sudo apt install -y openjdk-17-jdk

# 或者手动安装Adoptium JDK
wget https://github.com/adoptium/temurin17-binaries/releases/download/jdk-17.0.9%2B11/OpenJDK17U-jdk_x64_linux_hotspot_17.0.9_11.tar.gz
tar -xzf OpenJDK17U-jdk_x64_linux_hotspot_17.0.9_11.tar.gz -C /usr/local/
sudo ln -s /usr/local/jdk-17.0.9+11 /usr/local/jdk17

# 配置环境变量
echo 'export JAVA_HOME=/usr/local/jdk17' >> ~/.bashrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 验证
java -version
```

**下载地址**: <https://maven.apache.org/download.cgi>

**安装步骤**:

1. 下载 `apache-maven-3.9.6-bin.zip`
2. 解压到 `C:\Program Files\Apache\maven` 或自定义路径


```XML
  <mirrors>
    <mirror>
      <id>aliyunmaven</id>
      <mirrorOf>*</mirrorOf>
      <name>阿里云公共仓库</name>
      <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
  </mirrors>

  <profiles>
    <profile>
      <id>jdk-17</id>
      <activation>
        <activeByDefault>true</activeByDefault>
        <jdk>17</jdk>
      </activation>
      <properties>
        <maven.compiler.source>17</maven.compiler.source>
        <maven.compiler.target>17</maven.compiler.target>
        <maven.compiler.compilerVersion>17</maven.compiler.compilerVersion>
      </properties>
    </profile>
  </profiles>

```

1. **验证安装**:


#### 4.1.4 MySQL 8.0 安装和配置

##### Windows环境

**下载地址**: <https://dev.mysql.com/downloads/installer/>

**安装步骤**:

1. 下载MySQL Installer（社区版）
2. 运行安装程序，选择"Server only"或"Full"安装类型
3. **关键配置项**:
   - **Type and Networking**: 选择"Development Computer"，端口保持默认 `3306`
   - **Authentication Method**: 选择强密码加密（推荐 `caching_sha2_password`）
   - **设置root密码**: 记住密码（示例: `Admin@123456`）
   - **Windows Service**: 勾选"Configure MySQL Server as a Windows Service"
4. **字符集配置**（重要！）:

编辑MySQL配置文件 `my.ini`（通常在 `C:\ProgramData\MySQL\MySQL Server 8.0\my.ini`）:

```ini
[mysqld]
# 字符集配置（支持中文和emoji）
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci

# 存储引擎
default-storage-engine=INNODB

# 连接配置
max_connections=500
wait_timeout=28800
interactive_timeout=28800

# SQL模式（宽松模式，兼容性更好）
sql_mode=STRICT_TRANS_TABLES,NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_ENGINE_SUBSTITUTION

# InnoDB缓冲池大小（根据内存调整，建议物理内存的50%-70%）
innodb_buffer_pool_size=1G

# 日志配置
log-error=mysql_error.log
slow_query_log=1
slow_query_log_file=mysql_slow.log
long_query_time=2

[client]
default-character-set=utf8mb4

[mysql]
default-character-set=utf8mb4
```

1. **重启MySQL服务使配置生效**:

```powershell
# 重启MySQL服务
net stop MySQL80
net start MySQL80

# 或者通过服务管理器重启
```

##### Linux环境 (CentOS 7/8/9)

```bash
# CentOS 7/8 - 安装MySQL 8.0
sudo yum localinstall https://dev.mysql.com/get/mysql80-community-release-el7-11.noarch.rpm
sudo yum install -y mysql-community-server

# 启动并设置开机自启
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 获取临时root密码
sudo grep 'temporary password' /var/log/mysqld.log

# 安全初始化（修改root密码等）
sudo mysql_secure_installation
```

##### Linux环境 (Ubuntu 20.04/22.04)

```bash
# 更新软件源并安装MySQL
sudo apt update
sudo apt install -y mysql-server

# 启动服务
sudo systemctl start mysql
sudo systemctl enable mysql

# 执行安全配置
sudo mysql_secure_installation
```

##### 字符集配置（Linux通用）

编辑 `/etc/mysql/my.cnf` 或 `/etc/my.cnf.d/server.cnf`:

```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
init_connect='SET NAMES utf8mb4'
```

重启MySQL:

```bash
sudo systemctl restart mysql
```

##### 创建数据库和用户

```sql
-- 登录MySQL（使用root账户）
mysql -u root -p

-- 创建数据库
CREATE DATABASE IF NOT EXISTS CityLibrary
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;

-- 创建专用应用用户（最小权限原则）
CREATE USER 'citylibrary'@'localhost' IDENTIFIED BY 'YourStrongPassword123!';
CREATE USER 'citylibrary'@'%' IDENTIFIED BY 'YourStrongPassword123!';

-- 授权访问CityLibrary数据库
GRANT ALL PRIVILEGES ON CityLibrary.* TO 'citylibrary'@'localhost';
GRANT ALL PRIVILEGES ON CityLibrary.* TO 'citylibrary'@'%';

-- 刷新权限
FLUSH PRIVILEGES;

-- 验证连接
SHOW DATABASES;
SELECT User, Host FROM mysql.user WHERE User = 'citylibrary';
```

> ⚠️ **安全提醒**:
>
> - 生产环境请使用强密码（至少12位，包含大小写字母、数字、特殊字符）
> - 不要使用root用户直接连接应用数据库
> - 定期更换数据库密码

### 4.2 数据库初始化

#### 4.2.1 创建数据库

数据库已在4.1.4节中创建完成。确认数据库存在：

#### 4.2.2 执行初始化脚本

**schema.sql 文件位置**:

```
spring-boot-backend/src/main/resources/schema.sql
```

**执行方式一：MySQL命令行**

```bash
# 方式A：完整导入（包含建库语句）
mysql -u root -p < spring-boot-backend/src/main/resources/schema.sql

# 方式B：仅导入表结构（如果数据库已创建）
mysql -u root -p CityLibrary < spring-boot-backend/src/main/resources/schema.sql

# 使用应用用户导入（推荐）
mysql -u citylibrary -p CityLibrary < spring-boot-backend/src/main/resources/schema.sql
```

**执行方式二：MySQL客户端工具**

- **Navicat Premium**: 右键数据库 → 运行SQL文件 → 选择schema.sql
- **DBeaver**: 右键数据库 → SQL编辑器 → 打开文件 → 执行
- **MySQL Workbench**: File → Open SQL Script → 选择schema.sql → Execute
- **phpMyAdmin**: 导入标签页 → 选择文件 → 执行

**Flyway自动迁移说明**

本项目使用Spring Boot内置的SQL初始化机制（非Flyway），但提供了手动迁移脚本：

**迁移脚本位置**: `spring-boot-backend/src/main/resources/migration/`

| 脚本文件                                       | 版本    | 功能描述                                                                               |
| ------------------------------------------ | ----- | ---------------------------------------------------------------------------------- |
| `V1.0.1__fix_spelling_errors_rollback.sql` | 1.0.1 | 修复拼写错误回滚                                                                           |
| `V1.0.2__create_missing_tables.sql`        | 1.0.2 | 创建缺失的4张表（book\_review, book\_import\_detail, user\_behavior, book\_recommendation） |
| `V1.0.3__create_user_favorite_table.sql`   | 1.0.3 | 创建用户收藏表(user\_favorite)                                                            |
| `V1.0.4__add_missing_columns.sql`          | 1.0.4 | 为notification/notification\_template/reading\_room等表添加缺失字段                         |

**迁移执行顺序**（按版本号升序）:

```bash
# 依次执行迁移脚本
cd spring-boot-backend/src/main/resources/migration/

mysql -u citylibrary -p CityLibrary < V1.0.2__create_missing_tables.sql
mysql -u citylibrary -p CityLibrary < V1.0.3__create_user_favorite_table.sql
mysql -u citylibrary -p CityLibrary < V1.0.4__add_missing_columns.sql
```

> **注意**: 如果使用完整的 `schema.sql` 初始化，则无需再单独执行迁移脚本（schema.sql已包含最新表结构）。

**完整Nginx配置示例** (`/etc/nginx/conf.d/citylibrary.conf` 或 `nginx/conf/citylibrary.conf`):

```nginx
# ============================================================
# 城市图书馆管理系统 - Nginx配置
# 文件位置: /etc/nginx/conf.d/citylibrary.conf
# 用途: 前端静态资源托管 + API反向代理 + SSL/HTTPS
# ============================================================

# 上游服务器定义（后端API集群）
upstream citylibrary_backend {
    server 127.0.0.1:8080 weight=1 max_fails=3 fail_timeout=30s;
    # 如有多台后端，可添加:
    # server 192.168.1.101:8080 weight=1;
    # server 192.168.1.102:8080 weight=1;

    # 长连接配置
    keepalive 32;
}

# HTTP -> HTTPS 重定向（强制HTTPS）
server {
    listen 80;
    listen [::]:80;
    server_name your-domain.com www.your-domain.com your-ip;

    # Let's Encrypt证书验证
    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    # 其他所有请求重定向到HTTPS
    location / {
        return 301 https://$host$request_uri;
    }
}

# HTTPS主配置
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name your-domain.com www.your-domain.com your-ip;

    # ==================== SSL/TLS配置 ====================
    # SSL证书路径（Let's Encrypt免费证书或其他CA证书）
    ssl_certificate     /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # SSL协议和加密套件（安全配置）
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;

    # SSL会话缓存（提升性能）
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 1d;
    ssl_session_tickets off;

    # OCSP Stapling（提升SSL握手性能）
    ssl_stapling on;
    ssl_stapling_verify on;
    ssl_trusted_certificate /etc/letsencrypt/live/your-domain.com/chain.pem;

    # HSTS（HTTP严格传输安全，告诉浏览器只使用HTTPS）
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;

    # 安全头配置
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' data: blob:; font-src 'self'; connect-src 'self' ws: wss:;" always;

    # ==================== 前端静态资源托管 ====================
    root /var/www/citylibrary;
    index index.html;

    # Gzip压缩配置（减少传输体积60%+）
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_min_length 1000;
    gzip_types
        text/plain
        text/css
        text/javascript
        application/javascript
        application/json
        application/xml
        application/rss+xml
        image/svg+xml
        font/ttf
        font/otf
        application/vnd.ms-fontobject;

    # 缓存策略配置
    # HTML文件不缓存（确保更新及时）
    location = /index.html {
        add_header Cache-Control "no-cache, no-store, must-revalidate";
        add_header Pragma "no-cache";
        add_header Expires "0";
        try_files $uri $uri/ /index.html;
    }

    # 静态资源文件（JS/CSS/图片等）长期缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
        access_log off;

        # 静态资源文件直接由Nginx提供，不走后端
        try_files $uri =404;
    }

    # 上传文件目录（如果Nginx直接托管）
    location /uploads/ {
        alias /data/citylibrary/uploads/;
        expires 7d;
        add_header Cache-Control "public";

        # 限制请求大小（与后端配置一致）
        client_max_body_size 50m;

        # 防止目录遍历攻击
        autoindex off;
    }

    # ==================== API反向代理 ====================
    location /api/ {
        proxy_pass http://citylibrary_backend;

        # 传递真实客户端信息
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Port $server_port;

        # WebSocket支持（如果使用WebSocket）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        # 超时配置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 120s;

        # 缓存配置（API响应不缓存）
        add_header Cache-Control "no-store";

        # 请求体大小限制（与后端multipart配置匹配）
        client_max_body_size 100m;

        # 错误页面定制
        proxy_intercept_errors off;
    }

    # 健康检查端点（负载均衡器和监控使用）
    location /health {
        access_log off;
        return 200 '{"status":"ok"}';
        add_header Content-Type application/json;
    }

    # ==================== SPA路由支持 ====================
    # Vue Router history模式下，所有前端路由都返回index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # ==================== 日志配置 ====================
    access_log /var/log/nginx/citylibrary_access.log combined buffer=32k flush=5m;
    error_log  /var/log/nginx/citylibrary_error.log warn;

    # ==================== 速率限制（防暴力破解） ====================
    # 定义限制区域（针对登录接口）
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;
    limit_req_zone $binary_remote_addr zone=login_limit:10m rate=3r/m;

    # 登录接口特殊限制
    location ~* /api/(auth/login|auth/register) {
        limit_req zone=login_limit burst=5 nodelay;
        proxy_pass http://citylibrary_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 自定义错误页面
    error_page 500 502 503 504 /50x.html;
    location = /50x.html {
        root /usr/share/nginx/html;
    }
}
```

### 5.1 application.properties完整配置项说明

**关键配置项注意事项**:

1. **`spring.datasource.hikari.maximum-pool-size=1`**:
   - ⚠️ **当前默认值为1，这在生产环境中严重不足！**
   - 开发环境可能够用，但生产环境会导致严重的性能瓶颈
   - **建议**: 生产环境设置为 10-50（根据并发量调整）
2. **`jwt.secret`**:
   - ⚠️ **当前为示例密钥，绝对不能用于生产环境！**
   - 必须替换为随机生成的≥64字节的强密钥
   - 泄露该密钥将导致所有Token可被伪造
3. **`upload.path=uploads`**:
   - 相对路径，相对于项目根目录（即 `spring-boot-backend/uploads/`）
   - 生产环境建议使用绝对路径，便于备份和权限管理

#### 日志轮转配置

**Logback配置**（`src/main/resources/logback-spring.xml`, 如不存在则创建）:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    <!-- 定义日志文件路径 -->
    <property name="LOG_PATH" value="${LOG_PATH:-logs}"/>
    <property name="LOG_FILE" value="${LOG_FILE:-citylibrary}"/>

    <!-- 控制台输出 -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- 文件滚动输出 -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>${LOG_PATH}/${LOG_FILE}.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
            <!-- 按日期和大小滚动 -->
            <fileNamePattern>${LOG_PATH}/${LOG_FILE}.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
            <!-- 单个文件最大100MB -->
            <maxFileSize>100MB</maxFileSize>
            <!-- 保留30天的日志 -->
            <maxHistory>30</maxHistory>
            <!-- 总大小上限3GB -->
            <totalSizeCap>3GB</totalSizeCap>
        </rollingPolicy>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{50} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- 异步日志（提升性能） -->
    <appender name="ASYNC_FILE" class="ch.qos.logback.classic.AsyncAppender">
        <queueSize>1000</queueSize>
        <discardingThreshold>0</discardingThreshold>
        <neverBlock>true</neverBlock>
        <appender-ref ref="FILE"/>
    </appender>

    <!-- 日志级别配置 -->
    <root level="INFO">
        <appender-ref ref="CONSOLE"/>
        <appender-ref ref="ASYNC_FILE"/>
    </root>

    <!-- 特定包级别 -->
    <logger name="com.example.demo" level="DEBUG" additivity="false">
        <appender-ref ref="ASYNC_FILE"/>
    </logger>
</configuration>
```

**关键JVM指标及调优建议**:

| 指标             | 正常范围    | 告警阈值        | 优化措施              |
| -------------- | ------- | ----------- | ----------------- |
| **堆内存使用率**     | 40%-70% | >85%        | 增加 `-Xmx` 或排查内存泄漏 |
| **Young GC频率** | <5次/秒   | >10次/秒      | 调整新生代大小或Eden区比例   |
| **Full GC频率**  | <1次/小时  | >1次/分钟      | 严重问题，需排查大对象或内存泄漏  |
| **GC暂停时间**     | <100ms  | >500ms      | 调整G1GC目标暂停时间或堆大小  |
| **线程数量**       | <500    | >1000       | 排查线程泄漏或线程池配置不当    |
| **CPU使用率**     | <70%    | >90%（持续5分钟） | 排查CPU密集型操作或增加节点   |

```bash
#!/bin/bash
# 数据库备份脚本 - backup_database.sh
# 建议配置为cron定时任务

BACKUP_DIR="/data/backups/citylibrary/db"
```

#### **问题6：内存溢出OOM (OutOfMemoryError)**

**症状表现**:

- 应用突然崩溃退出
- 日志显示 `java.lang.OutOfMemoryError: Java heap space` 或 `Metaspace`
- 系统变慢，频繁Full GC

**可能原因及排查步骤**:

| 原因              | 排查方法                                                         | 解决方案                                                 |
| --------------- | ------------------------------------------------------------ | ---------------------------------------------------- |
| **堆内存过小**       | 查看启动参数 `-Xmx`                                                | 增大 `-Xmx`（建议物理内存的50%-70%）                            |
| **内存泄漏**        | 使用jvisualvm/jprofiler分析heap dump                             | 找出持有大量对象的类修复泄漏代码（如未关闭的连接、集合无限增长）                     |
| **大对象分配**       | 查找 `OutOfMemoryError: Requested array size exceeds VM limit` | 优化大数据量查询（分页）、避免一次性加载全部数据                             |
| **Metaspace溢出** | 查找 `Metaspace` 相关OOM                                         | 增大 `-XX:MaxMetaspaceSize=512m`检查动态类加载（如反射、CGLIB代理过多） |
| **线程栈溢出**       | 查找 `StackOverflowError`                                      | 增大 `-Xss`（线程栈大小）检查递归调用深度                             |

**OOM排查步骤**:

```bash
# 1. 开启OOM时自动Dump（添加到启动参数）
-XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=/data/citylibrary/dumps/

# 2. 当发生OOM后，分析dump文件
# 使用Eclipse MAT (Memory Analyzer Tool)
mat /data/citylibrary/dumps/java_pidXXXX.hprof

# 或使用jvisualvm
jvisualvm # File → Load → 选择hprof文件

# 3. 查看内存使用趋势
jstat -gc <pid> 5000  # 每5秒输出一次GC统计

# 4. 触发手动Dump（在内存增长异常时）
jmap -dump:format=b,file=manual_dump.hprof <pid>

# 5. 查看对象分布（找出内存大户）
jmap -histo <pid> | head -30
```

**JVM内存优化参数参考**:

```bash
# 生产环境推荐配置（8GB物理内存服务器）
java -Xms1g \                    # 初始堆内存1GB
     -Xmx4g \                    # 最大堆内存4GB（不超过物理内存的70%）
     -XX:MetaspaceSize=256m \    # Metaspace初始256MB
     -XX:MaxMetaspaceSize=512m \ # Metaspace最大512MB
     -XX:+UseG1GC \              # 使用G1垃圾收集器（低延迟）
     -XX:MaxGCPauseMillis=200 \  # GC暂停目标200ms
     -XX:+HeapDumpOnOutOfMemoryError \  # OOM时自动Dump
     -XX:HeapDumpPath=/data/citylibrary/dumps/ \  # Dump文件路径
     -XX:+ExitOnOutOfMemoryError \  # OOM时退出（便于systemd重启）
     -XX:+PrintGCDetails \        # 打印GC详情
     -XX:+PrintGCDateStamps \     # 打印GC时间戳
     -Xloggc:/data/citylibrary/logs/gc.log \  # GC日志文件
     -jar demo.jar
```

```sql
FROM information_schema.tables

ORDER BY Size_MB DESC;

SELECT
    table_name,
    ROUND(data_length / 1024 / 1024, 2) AS 'Data_MB',
    ROUND(index_length / 1024 / 1024, 2) AS 'Index_MB',
    table_rows
FROM information_schema.tables
WHERE table_schema = 'CityLibrary'
ORDER BY data_length DESC
LIMIT 20;

-- 碎片整理（碎片率>20%建议OPTIMIZE）

    table_name,
    ROUND(data_free / 1024 / 1024, 2) AS 'Fragmentation_MB',
    ROUND(data_free / data_length * 100, 2) AS 'Fragmentation_Pct'
FROM information_schema.tables

AND data_free > 0
```

## 9. 安全加固建议

### 9.1 密码策略

**强制复杂密码规则**:

```sql
-- MySQL用户密码策略（推荐设置为MEDIUM或STRONG）
SHOW VARIABLES LIKE 'validate_password%';

-- 设置密码策略
SET GLOBAL validate_password.policy = MEDIUM;
SET GLOBAL validate_password.length = 12;
SET VALIDATE_PASSWORD.MIXED_CASE_COUNT = 1;
SET VALIDATE_PASSWORD.NUMBER_COUNT = 1;
SET VALIDATE_PASSWORD.SPECIAL_CHAR_COUNT = 1;
```

**应用层密码强度要求**:

| 要求项       | 规则                                    |
| --------- | ------------------------------------- |
| **最小长度**  | ≥12位（推荐16位以上）                         |
| **字符种类**  | 必须包含：大写字母、小写字母、数字、特殊字符                |
| **禁止弱密码** | 123456、password、admin123、qwerty等常见弱密码 |
| **定期更换**  | 数据库密码：每90天JWT密钥：每年或疑似泄露时系统管理员密码：每60天  |
| **禁止复用**  | 最近5次使用过的密码不能重复                        |
| **锁定策略**  | 连续失败5次锁定30分钟                          |

**密码存储安全**:

- ✅ 用户密码使用BCrypt加密存储（本项目已采用）
- ❌ 明文存储（绝对禁止）
- ❌ MD5/SHA1单向哈希（已被破解，不安全）
- ✅ BCrypt/Argon2/PBKDF2（推荐）

### 9.5 应用安全

#### SQL注入防护

本项目采用 **MyBatis-Plus ORM框架**，天然具备SQL注入防护能力：

```java
// ✅ 安全：参数化查询（MyBatis-Plus默认行为，推荐）
@Select("SELECT * FROM user_account WHERE username = #{username}")
UserAccount findByUsername(@Param("username") String username);

// ❌ 危险：字符串拼接（绝对禁止！）
// @Select("SELECT * FROM user_account WHERE username = '" + username + "'")

// ✅ 安全：MyBatis-Plus QueryWrapper（类型安全）
QueryWrapper<UserAccount> wrapper = new QueryWrapper<>();
wrapper.eq("username", username);
userMapper.selectOne(wrapper);
```

#### XSS攻击防护

- **前端**: Vue3自动进行HTML转义（`{{ }}` 双花括号默认转义）
- **后端**: Spring Security内置XSS过滤器
- **额外防护**: 可集成 OWASP Java HTML Sanitizer 库

#### CSRF防护

- **RESTful API + JWT架构**: 天然免疫CSRF攻击（因为JWT存储在Header而非Cookie中）
- **如使用Session-based认证**: 必须启用Spring Security的CSRF保护

### 10.1 版本升级步骤

**标准升级流程**（适用于小版本迭代）:

```bash
# ===== 1. 备份当前版本 =====
# 备份数据库（见7.3节）
/opt/scripts/backup_database.sh

cp /opt/citylibrary/backend/application.properties /opt/citylibrary/backups/config_$(date +%Y%m%d).properties.bak

tar -czvf /opt/citylibrary/backups/uploads_$(date +%Y%m%d).tar.gz /opt/citylibrary/uploads/


# ===== 2. 停止服务 =====
python launcher.py stop
# 或
sudo systemctl stop citylibrary


# ===== 3. 部署新版本 =====
# 方式A：从Git拉取新代码
cd /opt/citylibrary/backend
git pull origin main
mvn clean package -DskipTests

# 方式B：上传新的JAR包
scp new-version.jar user@server:/opt/citylibrary/backend/target/demo-0.0.1-SNAPSHOT.jar


# ===== 4. 执行数据库迁移（如果有） =====
cd /opt/citylibrary/backend/src/main/resources/migration/

# 查看待执行的迁移脚本
ls -la V*.sql | sort

# 按顺序执行迁移脚本
for sql_file in $(ls V*.sql | sort); do
    echo "执行迁移: $sql_file"
    mysql -u citylibrary -p CityLibrary < $sql_file
done


# ===== 5. 启动服务并验证 =====
python launcher.py start
# 或
sudo systemctl start citylibrary


# ===== 6. 验证升级结果 =====
# 健康检查
curl http://localhost:8080/api/test/health

# 功能验证（参考4.4.5节的验证清单）
# - 页面正常加载
# - 登录功能正常
# - 核心业务功能正常

# 回滚方案（如果升级失败）:
# 1. 停止服务
# 2. 恢复数据库备份
# 3. 恢复旧版本JAR包
# 4. 启动服务
# 5. 验证回滚后功能正常
```
